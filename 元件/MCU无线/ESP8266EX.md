# ESP8266EX

## 1. 身份选型

- **型号**: ESP8266EX
- **内核**: Tensilica L106 32-bit RISC，80 MHz（最大 160 MHz）
- **无线**: WiFi 802.11 b/g/n，**仅 HT20（20 MHz 带宽）**，2.4 GHz 频段（2412–2484 MHz），**无蓝牙**
- **存储架构**: 片内**无用户可编程 Flash**，外挂 SPI Flash 为**必选**（1–16 MB）；最少 512 kB（无 OTA）/ 1 MB（带 OTA）
- **推荐升级型号**: [[ESP32-C6]] / ESP8684（乐鑫官方建议用 ESP8684 替代新设计）
- **车规**: —

## 2. 极限工况

- **供电电压 (VDD)**: 2.5 – 3.6 V
- **工作温度**: –40 ～ +125 °C
- **ESD 耐受**: HBM ±2 kV / CDM ±0.5 kV
- **最大焊接温度**: 260 °C（IPC/JEDEC J-STD-020）

## 3. 推荐工作条件

- **VDD（模拟/功放/数字）**: 3.3 V 典型，2.5–3.6 V
- **VDDPST（IO 电源）**: 1.8–3.6 V
- **晶振频率**: 24–52 MHz，频率容限 ±15 ppm，温度漂移 ±15 ppm（–25 ～ +75 °C）
- **外部时钟振幅**: 0.8–1.5 Vpp
- **上电时序**: VDD 上升时间 10–2000 μs，EXT_RSTB 和 CHIP_EN 依次在 VDD 稳定后至少 0.1 ms 拉高

## 4. 功耗热特性

| 模式 | 条件 | 典型电流 | 说明 |
|------|------|----------|------|
| Active TX 802.11b | CCK 11 Mbps, POUT=+17 dBm | 170 mA | 50% 占空比 |
| Active TX 802.11g | OFDM 54 Mbps, POUT=+15 dBm | 140 mA | |
| Active TX 802.11n | MCS7, POUT=+13 dBm | 120 mA | |
| RX 802.11b/g/n | 1024 B 包长 | 50–56 mA | |
| Modem-sleep | CPU 运行，WiFi 关闭 | 15 mA | DTIM3 间隔 300 ms |
| Light-sleep | CPU/外设暂停 | 0.9 mA | 保持 WiFi 连接 |
| Deep-sleep | 仅 RTC 工作 | **20 μA** | 2.5 V 下测得 |
| 关闭 | — | 0.5 μA | |

> [!warning] Deep-sleep 经典坑
> Deep-sleep 唤醒需要将 **XPD_DCDC (GPIO16, Pin 8)** 外部连接到 **EXT_RSTB (Pin 32)**。RTC 定时器到期后 XPD_DCDC 输出高电平触发复位唤醒——如果 PCB 上未布这根飞线，Deep-sleep 将无法自动唤醒。该引脚在正常运行时**必须悬空**（不可被外部拉高/拉低）。

### 三种睡眠模式的选择逻辑

- **Modem-sleep**（15 mA）：CPU 必须持续运行的场景（PWM 调光、I2S 音频）。保持 WiFi 关联，无数据时按 802.11 U-APSD 关闭射频，DTIM3 下睡 300 ms、醒 3 ms 收 Beacon
- **Light-sleep**（0.9 mA）：CPU 可暂停的场景（WiFi 开关类设备），比 Modem-sleep 多暂停了 CPU，唤醒源为 MAC/主机/RTC 定时器/外部中断
- **Deep-sleep**（20 μA）：长周期上报（如每 100 s 测一次温度的传感器）。每次醒来需 **0.3–1 s** 重连 AP；以 300 s 周期计，整体平均电流 **< 1 mA**

## 5. IO 接口特性

- **GPIO**: 17 个，双向三态；**仅可内部上拉**（XPD_DCDC 仅可内部下拉），单管脚最大驱动 **12 mA**；IO 电平跟随 VDDPST
  - VIL: –0.3 ～ 0.25×VIO / VIH: 0.75×VIO ～ 3.6 V
  - VOL: 0 ～ 0.1×VIO / VOH: 0.8×VIO ～ VIO
- **ADC**: 1 × 10-bit SAR ADC，通过 TOUT（Pin 6）输入，**两种模式互斥不可同时使用**：
  - **测量 VDD3P3**：TOUT 悬空，vdd33_const = 0xFF，射频自动据此优化
  - **测量外部 0–1.0 V 信号**：TOUT 接外部电路，必须在初始化数据中设置正确的 vdd33_const
- **UART**: 2 路；UART0 + 硬件流控（RTS/CTS），UART1 **仅 TX（GPIO2）** 常用作 log
  - 上电打印波特率：40 MHz 晶振 → 115200，26 MHz 晶振 → 74880
  - 若需屏蔽上电打印，可将 U0TXD/U0RXD 与 U0RTS/U0CTS 交换
- **SPI**: 2 个（SPI + HSPI），均可为主/从
  - SPI 主机模式时钟 ≤ **80 MHz**，从机 ≤ 20 MHz
  - HSPI 时钟 ≤ **20 MHz**（主/从）
- **SDIO**: 1 个从机接口，SDIO v1.1 (25 MHz) / v2.0 (50 MHz)，1/4-bit SD + SPI 模式
- **I2C**: **软件模拟**（非硬件外设），通过 GPIO14 (SCL) + GPIO2 (SDA)，≤ 100 kHz
- **I2S**: 1 入 1 出，支持链表 DMA；用途包括音频采集与传输、**驱动 LED 彩灯（WS2812 系列）**
- **PWM**: **软件实现**（定时器中断），4 路输出，最小分辨率 44 ns，频率 100 Hz–1 kHz
- **IR**: 1 路红外遥控，软件实现，NEC 编码，38 kHz 载波，传输距离 ~1 m

> [!warning] 单 ADC 的核心限制
> ESP8266 仅有 **1 个 ADC 通道**，无法同时测量电源电压和外部模拟信号。采用外部测量模式时 TOUT 输入不得超过 1.0 V，且必须在 `esp_init_data_default.bin` 第 107 byte 正确设置 vdd33_const，否则射频校准会基于错误电压导致性能下降。

## 6. 核心功能

### 6.1 CPU 与存储架构

Tensilica L106 是**无片内用户 Flash** 的 MCU 内核，所有用户程序必须从外部 SPI Flash 加载。片内 SRAM 在 Station 模式连接 AP 后用户可用约 **50 kB**（Heap + Data）。

架构含义：
- SPI Flash **不是可选件，是必选外围**——无 Flash 芯片无法运行任何用户代码
- Flash 的 SPI 模式（Standard/Dual/Quad）必须与 SDK 烧录时一致，否则无法启动
- OTA 升级需要 **≥ 1 MB** Flash 空间

### 6.2 WiFi 能力

| 项目 | 规格 |
|------|------|
| 标准 | 802.11 b/g/n，**仅 HT20**（20 MHz，无 HT40），2.4 GHz |
| 最高数据率 | 72.2 Mbps (MCS7) |
| 发射功率 | b: +20 dBm / g: +17 dBm / n: +14 dBm（可调） |
| 接收灵敏度 | 1 Mbps DSSS: –98 dBm / 11 Mbps CCK: –91 dBm / 54 Mbps OFDM: –75 dBm / MCS7: –72 dBm |
| 工作模式 | Station / SoftAP / SoftAP+Station / 混杂模式 |
| 安全加密 | WPA/WPA2, WEP/TKIP/AES，硬件 CCMP/TKIP/WEP/CRC 卸载 |
| 硬件加速 | Beacon 自动监测（硬件 TSF）、重组、RTS/CTS、块回复 |
| 天线选项 | PCB 板载、外置、IPEX、陶瓷贴片 |
| 输出阻抗 | **39 + j6 Ω**（非 50 Ω），必须设计 π 型匹配网络 |

### 6.3 启动模式（Strapping）

上电瞬间 GPIO0、GPIO2、MTDO（GPIO15）被采样以决定启动方式：

| GPIO15 (MTDO) | GPIO0 | GPIO2 | 启动模式 |
|---------------|-------|-------|----------|
| 0 | 1 | 1 | **Flash 启动（正常运行）** |
| 0 | 0 | 1 | **UART 下载（固件烧录）** |
| 1 | × | × | SDIO 从机启动（挂载到主机 MCU） |

ROM 启动日志中 `boot mode:(3,x)` 的首位数字即 `(GPIO15<<2)|(GPIO0<<1)|GPIO2`——3 为 Flash 启动，1 为 UART 下载。

> [!warning] 启动选脚必须正确处理
> - GPIO0：正常运行接上拉（10 kΩ 到 VDD），下载时按键拉低
> - GPIO15 (MTDO)：**必须外部下拉到地**（片内无下拉），否则芯片无法启动
> - GPIO2：必须上拉（推荐 10 kΩ）
> - U0TXD 在上电期间应避免被外部电路下拉
> - SDIO 总线引脚（Pin 18–23）需串联 **200 Ω** 电阻

## 7. 引脚典型连线

### 最小系统要点

- **电源**: VDDA / VDD3P3 / VDDPST 统一供 3.3 V；VDD_RTC（Pin 5）悬空
- **天线**: LNA（Pin 2）输出阻抗 39 + j6 Ω，**必须保留 π 型匹配网络**，不可直接接 50 Ω 天线
- **晶振**: 40 MHz 或 26 MHz，串联电阻 ≤ 65 Ω，负载电容 ≤ 32 pF
- **复位**: EXT_RSTB（Pin 32）接 10 kΩ 上拉 + 按键到地
- **使能**: CHIP_EN（Pin 7）接上拉使能
- **RES12K**: Pin 31 **必须**串联 12 kΩ（1%）电阻到地——**不可省略，不可改用其他阻值**

### Deep-sleep 唤醒连线

```
XPD_DCDC (GPIO16, Pin 8) ——— 外部短接 ——— EXT_RSTB (Pin 32)
```

> [!warning] 如 PCB 上未走此连线，则 deep-sleep 模式无法自动唤醒，只能通过外部复位或重新上电。

### 电源设计要求

- **峰值电流**: 802.11b TX 时典型 170 mA，平均工作电流 80 mA；电源应按 **≥ 500 mA** 裕量设计（覆盖发射突发 + Flash 读取叠加）
- **纹波敏感**: 射频校准以电源电压为基准（vdd33_const），电压误差容限仅 **±0.2 V**；VDD3P3 纹波直接调制 PA 输出，恶化 EVM 和频谱模板。TX 突发（ms 级脉冲电流）会在电源阻抗上产生压降——VDD3P3 引脚旁需要**大容量储能电容（≥ 10 μF）+ 0.1 μF 高频去耦**，紧贴 Pin 3/4 放置
- **LDO vs DC-DC**: 优先 LDO（如 AMS1117 等 800 mA 级）换取干净电源；若用 DC-DC 供电，开关频率纹波必须充分滤除，且注意 DC-DC 电感对射频的辐射干扰

### 射频布局要点

- LNA 走线按 **39 + j6 Ω → 50 Ω** 阻抗变换设计 π 型匹配（两并联电容 + 一串联电感的占位必须保留，实测后定值）
- 天线馈线做 50 Ω 受控阻抗微带线，参考层完整，禁止跨分割
- 天线净空区内（板载天线正下方及周边）**禁止铺铜、走线、放器件**
- 晶振远离天线与 PA 区域；晶振质量直接决定相位噪声与 WiFi 灵敏度（要求 ±15 ppm）
- 手册明确 ESP8266EX 已内置蜂窝/蓝牙/DDR/LVDS/LCD 干扰消除机制，但强干扰源（摄像头 LVDS、DDR 走线）仍应远离射频区

## 9. 代际对比

| 对比项 | ESP8266EX | [[ESP32-S3]] | [[ESP32-C6]] |
|--------|-----------|--------------|--------------|
| 内核 | Tensilica L106 单核 32-bit | Xtensa LX7 双核 32-bit | RISC-V 单核 32-bit |
| 最大主频 | 160 MHz | 240 MHz | 160 MHz |
| 片内用户 Flash | **无**（外挂必选） | 有（叠封） | 有（叠封） |
| 可用 SRAM | ~50 kB | 512 kB | 512 kB |
| WiFi | b/g/n HT20 2.4G | b/g/n HT40 2.4G | b/g/n HT20 2.4G |
| 蓝牙 | **无** | BLE 5.0 | BLE 5.0 |
| 802.15.4 / Thread / Zigbee | 无 | 无 | **有** |
| ADC | 1 × 10-bit | 2 × 12-bit | 1 × 12-bit |
| 硬件 I2C | **无**（软件模拟） | 有 | 有 |
| USB OTG | 无 | 有 | 无 |
| 安全引擎 | WPA/WPA2 | AES/SHA/RSA/ECC 全套 | 完整安全子系统 |
| 生态成熟度 | 极高（2014–） | 高 | 中（2023–） |

> [!note] 2026 年的选型建议
> ESP8266 仅在**极致低成本**（模组级 < $1.5）、**纯 WiFi 开关/传感器**、且无需蓝牙或 Thread 的维护型项目中有意义。新设计优先考虑 [[ESP32-C6]]（Matter/Zigbee 就绪，未来主力）或 [[ESP32-S3]]（高性能 + BLE）。ESP8266 的优势在于生态极度成熟、中文资料丰富、学习曲线几乎为零。
