# ICM-42688-P

> TDK InvenSense 高精度 6 轴 MEMS IMU：3 轴陀螺仪 + 3 轴加速度计，2.8 mdps/√Hz 陀螺噪声为消费级最低档，±5 mdps/°C 零偏温漂、2 kB FIFO（20-bit 高分辨率包）、APEX 运动引擎、31–50 kHz 外部时钟（CLKIN）同步。定位就是手册第一页写的：HMD & Robotics。
> 依据：DS-000347 Rev 1.2（2020-04-19，109 页）。

> [!note] 选型要点
> -P 后缀是家族中的"单接口"（Single-Interface）版本：**只有 UI（User Interface）主机路径，没有 AUX/OIS 辅助接口路径**（那是带光学防抖输出的兄弟型号才有的），引脚 2/3/10/11 全部 RESV。它用省下来的资源换了两件事：最低噪声档 + RTC 外部时钟输入——正是多节点动捕/HMD 最需要的两个特性。

### 3.3 这些数字对姿态解算意味着什么

把手册值翻译成滤波器工程师的语言（以下换算为推导值，非手册原文）：

- **角度随机游走（ARW）**：陀螺噪声密度 0.0028 dps/√Hz × 60 ≈ **0.168 °/√h**。纯积分 1 小时的姿态角随机误差标准差约 0.17°——这是白噪声下限，任何 [Madgwick](../../知识/姿态解算/梯度下降姿态解算.md)/[ESKF](../../知识/姿态解算/误差状态卡尔曼滤波.md) 都无法低于它，只能靠加计/磁力计观测把长期漂移拉回来。
- **RMS 噪声随带宽缩放**：σ ≈ 噪声密度 × √NBW。100 Hz 带宽 0.028 dps；若开到 ODR/2 = 500 Hz 带宽则约 0.066 dps。**收窄 UI 滤波器带宽是免费的降噪，但代价是群延迟**（见第 6 节）。
- **零偏温漂才是主要敌人**：±5 mdps/°C 意味着芯片自热或环境升温 10 °C，零偏最多移动 0.05 dps——若不在线估计，航向角以约 3 °/min 的速度跑掉。这正是 [误差状态卡尔曼滤波](../../知识/姿态解算/误差状态卡尔曼滤波.md)、[VQF 姿态解算滤波器](../../知识/姿态解算/VQF%20姿态解算滤波器.md) 把陀螺零偏列入状态向量、[零速检测 ZUPT](../../知识/导航定位/零速检测%20ZUPT.md) 利用静止区间锁零偏的原因。手册**未给出 Allan 方差意义上的零偏不稳定性（Bias Instability）指标**，只给了温度系数——评估长期漂移需自行实测。
- **加计噪声与倾角**：0.65 mg-rms ≈ 0.037° 倾角噪声——足够精细；但 ±20 mg 初始零偏对应约 1.15° 倾角系统误差，**逐颗六面标定不可省**。
- 算法脉络与器件指标如何相互作用，参见 [IMU姿态解算算法演进](../../知识/姿态解算/IMU姿态解算算法演进.md)。

> [!note] 温度补偿实践
> 片上温度计与 IMU 同 ODR 更新，就是拿来做零偏-温度建模的：采集温度扫描曲线 → 拟合 bias(T) → 运行时用 `TEMP_DATA` 前馈补偿，再交给滤波器收残差。灵敏度温漂 ±0.005 %/°C 通常可忽略，零偏温漂不能。

> [!note] 模式切换的三条硬规则（Use Notes §12.1 + PWR_MGMT0）
> 1. **陀螺一旦开启至少保持 45 ms**；任何 OFF→ON 模式切换后 **200 µs 内禁止写寄存器**。
> 2. 加计 LP→LN 前 ODR 必须 ≥12.5 Hz；LN→LP 前 ODR 必须 ≤500 Hz。
> 3. 加计 LP 模式建议用 16x 平均滤波（官方驱动默认），以满足噪声要求并减小 LP↔LN 切换的零偏跳变。

## 5. I/O 接口

| 接口 | 速率 | 备注 |
|------|------|------|
| I3C | 12.5 MHz（SDR 12.5 Mbps / DDR 25 Mbps） | 支持带内中断 IBI、动态寻址、CRC |
| I²C | 1 MHz（Fast-mode Plus） | 地址 `110100X`：AD0=0 → **0x68**，AD0=1 → 0x69 |
| SPI | **24 MHz** | 3 线或 4 线；本项目使用 |

SPI 细节：MSB 先行，SCLK 上升沿锁存；默认 Mode 0/3（`DEVICE_CONFIG.SPI_MODE` 可切 Mode 1/2）；读操作首字节 = R/W 位(读=1) + 7 位寄存器地址，支持突发读；写为单字节写。上电即处于 SPI 模式；`INTF_CONFIG0.UI_SIFS_CFG` 可显式关掉不用的 SPI(10) 或 I²C(11)，避免总线毛刺误触发。输出摆率由 `DRIVE_CONFIG`（0x13）分接口配置（SPI/I3C 推荐值 5，I²C 推荐值 1）。

### 6.1 三级滤波各司其职

| 级 | 对象 | 作用 | 可配置项 |
|----|------|------|----------|
| 陷波 Notch | 仅陀螺，逐轴 | 压制 25–29 kHz 机械谐振峰（sense peak）泄漏到带内的噪声，同时保持低延迟 | 中心频率 1–3 kHz（出厂逐颗微调 COSWZ），带宽 8 档：1449/680/329/162/80/40/20/10 Hz |
| 抗混叠 AAF | 陀螺、加计独立 | 2 阶低通，在降采样前限带防混叠；用带宽换 RMS 噪声 vs 延迟 | 3dB 带宽 42–3979 Hz，由 DELT(1–63)/DELTSQR/BITSHIFT 三参数查表编程 |
| UI 滤波器 | 陀螺、加计独立 | 输出级低通，决定最终噪声带宽与群延迟 | 阶数 1/2/3 + 带宽 0–7/14/15 档 |

AAF 查表节选（DELT → 3dB 带宽）：1→42 Hz，6→258 Hz，13→585 Hz，21→997 Hz，32→1639 Hz，63→3979 Hz。

### 6.2 UI 滤波器带宽档位（LN 模式）

`GYRO/ACCEL_UI_FILT_BW`：0 = ODR/2；1 = max(400,ODR)/4（默认）；2 = /5；3 = /8；4 = /10；5 = /16；6 = /20；7 = /40；14/15 = 低延迟选项（Dec2 输出直接抽取）。

### 6.3 群延迟——直接吃掉姿态解算的相位裕度

ODR = 1 kHz 时的实测表（3dB 带宽 Hz / 直流群延迟 ms）：

| UI_FILT_BW 档 | 1 阶 | 2 阶 | 3 阶 |
|:---:|:---:|:---:|:---:|
| 0 (ODR/2) | 498.3 / 0.6 | 493.3 / 0.7 | 492.9 / 0.8 |
| 1 (默认) | 227.2 / 1.8 | 230.7 / 2.1 | 234.7 / 2.3 |
| 3 | 111.0 / 2.8 | 117.5 / 3.2 | 118.9 / 4.0 |
| 7 | 23.9 / 8.1 | 21.3 / 12.0 | 25.2 / 14.1 |

> [!tip] 滤波链取舍（动捕/HMD 视角）
> 姿态滤波器把陀螺样本当作"此刻"的角速度做积分——群延迟 δt 直接变成动态姿态滞后：ω = 500 dps 转动时，2 ms 延迟 ≈ 1° 瞬时姿态误差。快速运动追踪应保守收带宽（默认档 1，延迟 ~2 ms），把进一步降噪交给 [VQF 姿态解算滤波器](../../知识/姿态解算/VQF%20姿态解算滤波器.md) 这类算法侧低通（它显式建模了滤波延迟）；反之静态精度优先（如水平仪）可用档 7。**加计与陀螺应选同档带宽，否则两路观测相位不一致，融合时会打架。**

> [!note] 坑：Notch 与 AAF 只在 LN 模式下工作
> Use Notes §12.4 明确：陷波与抗混叠滤波器仅支持低噪声（LN）模式，主机有责任在开启它们期间保持 UI 路径处于 LN。LP 模式下走的是平均滤波器（1x/16x AVG），不要指望 AAF。

### 6.4 用户可编程偏置

标定结果可直接写进片内，省去 MCU 每样本减偏置：陀螺 ±64 dps、分辨率 1/32 dps；加计 ±1 g、分辨率 0.5 mg（Bank 4，OFFSET_USER0–8，12-bit/轴）。

## 7. FIFO 与数据读取

2048 字节硬件 FIFO，经 `FIFO_DATA`（0x30）串行读出，三种模式（`FIFO_CONFIG.FIFO_MODE`）：Bypass（默认关）/ Stream-to-FIFO（满后覆盖旧数据）/ STOP-on-FULL（满后丢新数据）。

### 7.1 四种包格式

| 包 | 触发条件 | 大小 | 内容 |
|----|----------|------|------|
| Packet 1 | 仅加计 | 8 B | Header + Accel XYZ(6) + Temp(1) |
| Packet 2 | 仅陀螺 | 8 B | Header + Gyro XYZ(6) + Temp(1) |
| Packet 3 | 加计+陀螺 | **16 B** | Header + Accel(6) + Gyro(6) + Temp(1, 8-bit) + Timestamp(2) |
| Packet 4 | FIFO_HIRES_EN=1 | **20 B** | Header + Accel(6) + Gyro(6) + Temp(2, 16-bit) + Timestamp(2) + 20-bit 扩展字节(3) |

Header 字节（1 B）关键位：bit7 = FIFO 空标志；bit6/5 = 含加计/陀螺；bit4 = 20-bit 扩展有效；bit3:2 = 时间戳类型（10 = ODR 时间戳，11 = FSYNC 时间）；bit1/0 = 加计/陀螺 ODR 相比上包发生了变化。

### 7.2 20-bit 高分辨率模式

Packet 4 的扩展 3 字节给每轴补 4 bit 低位：**陀螺实际 19 bit（LSB 恒 0），加计实际 18 bit（低 2 bit 恒 0）**；温度扩展为 16 bit。

> [!note] 坑：高分辨率模式强制满量程
> FIFO_HIRES_EN=1 时**只有 ±2000 dps / ±16 g 生效**——哪怕 FS_SEL 寄存器配了别的档也被忽略。对应灵敏度固定为 **131 LSB/dps** 与 **8192 LSB/g**。等效于"±2000 dps 量程 + ±250 dps 档的分辨率"，量程与分辨率兼得，代价是每包 20 B 的带宽。

### 7.3 水位中断与读取流程

- 水位 `FIFO_WM[11:0]`（0x60/0x61），单位由 `INTF_CONFIG0.FIFO_COUNT_REC` 决定（字节或记录）；达到即触发 FIFO_THS 中断。**FIFO_WM 禁止设 0**，且要先写水位再使能该中断源。`FIFO_WM_GT_TH` 可改为"只要 ≥ 水位就每个 ODR 都触发"。
- 读流程：读 `FIFO_COUNTH/L`（**必须读 COUNTL 才锁存两字节**，默认大端）→ 从 `FIFO_DATA` 突发读 N 字节 → 按 Header 解析。`FIFO_RESUME_PARTIAL_RD=1` 允许分段读续传，否则一包必须一次读完。
- 容量（换算）：16 B 包最多约 128 包（1 kHz ODR 下 ≈128 ms 缓冲），20 B 包约 102 包；手册要求**驱动侧一律按 2080 字节分配缓冲**（2048 + 读缓存余量）。丢包计数见 `FIFO_LOST_PKT0/1`。
- 无效样本标记：加计/陀螺 ODR 不同或传感器未就绪时，慢的那路会产生无效样本——`FIFO_HOLD_LAST_DATA_EN=0` 时以 **−32768**（16-bit）/ **−524288**（20-bit）作哨兵值填充；=1 时复制上一有效样本。解析器必须处理，否则会把哨兵当真值积分出一个巨大角速度尖峰。

> [!tip] 为什么用 FIFO 而不是逐样本 DRDY 中断
> 1 kHz × 16 B = 16 kB/s，攒 20 包（20 ms）再一次突发读，中断频率从 1 kHz 降到 50 Hz，SPI 事务开销和 MCU 唤醒次数都降一个量级——手册设计意图即"burst read then sleep"。时间戳字段（16-bit，默认 1 µs 分辨率，TMST_RES 可切 16 µs）让批量读出的样本仍可精确对齐时间轴。

## 8. 中断系统

- 两个中断脚 INT1（pin 4）、INT2（pin 9 复用），各自可配：推挽/开漏、脉冲/锁存、高/低有效（`INT_CONFIG` 0x14）。I3C 下还可走带内中断 IBI。
- 中断源自由路由：数据就绪、FIFO 水位、FIFO 满、AGC/PLL/复位完成、FSYNC（INT_SOURCE0/3），WoM XYZ、SMD、I3C 错误（INT_SOURCE1/4），APEX 事件（Bank4 INT_SOURCE6/7）。
- 状态读取：`INT_STATUS`（0x2D）/ `INT_STATUS2`（0x37，WoM/SMD）/ `INT_STATUS3`（0x38，APEX），读后自动清零；锁存模式的清除条件由 `INT_CONFIG0` 细配（读状态位 / 读传感器寄存器 / 读 FIFO）。

> [!note] 两个必改的默认值
> 1. `INT_CONFIG1.INT_ASYNC_RESET`（0x64 bit4）**默认为 1，必须写 0**，否则 INT1/INT2 引脚行为异常（Use Notes §12.6，驱动初始化经典遗漏项）。
> 2. ODR ≥ 4 kHz 时必须把中断脉宽从默认 100 µs 改为 8 µs（`INT_TPULSE_DURATION=1`）并禁用去断言时间（`INT_TDEASSERT_DISABLE=1`），否则脉冲比采样周期还长。

## 9. APEX 运动引擎（片上 DMP）

APEX = Advanced Pedometer and Event Detection – neXt gen，一颗只吃**加速度计**数据的小型 DMP，让 MCU 深睡时仍能识别手势。DMP 运行 ODR 仅 25/50 Hz 两档（`DMP_ODR`），加计 ODR 须为其整数倍且 ≥25 Hz。

| 功能 | 触发/输出 | 关键参数 |
|------|-----------|----------|
| 计步器 Pedometer | STEP_CNT 16-bit 计数 + 步频 STEP_CADENCE + 活动分类（走/跑）+ 步检测中断 | 阈值组 Bank4 0x40–0x42；溢出 65535 回绕并发 STEP_CNT_OVF 中断 |
| 倾斜检测 Tilt | 倾角超过 **35°** 持续设定时间后中断 | 等待时间 0/2/4/6 s |
| 抬腕 Raise to Wake/Sleep | 抬起/放下手势各发中断 | 睡眠超时 1.28–10.24 s；安装矩阵 MOUNTING_MATRIX 8 种 |
| 敲击检测 Tap | 单击/双击、轴向、极性、双击间隔 | 加计 ODR 限 200 Hz(低功耗)/500 Hz(常规)/1 kHz(高性能)；>1 kHz 不可用 |
| 运动唤醒 WoM | 任一/全部轴超阈值中断 | 阈值/轴，0–1 g，分辨率 1g/256 ≈ **3.9 mg** |
| 显著运动 SMD | 两次 WoM 间隔 1 s（短）或 3 s（长）内先后出现 | SMD_MODE = 10/11 |

- **DMP Power Save**（APEX_CONFIG0 bit7，默认开）：无 WoM 事件时 APEX 挂起，被 WoM 唤醒后再跑设定时长（0–60 s）——用它必须同时启用 WoM。
- 初始化套路（手册流程共性）：配加计 ODR/LP 模式 → `DMP_MEM_RESET_EN` → 写 Bank4 参数 → `DMP_INIT_EN` → 路由中断源 → 等 50 ms → 使能对应 ENABLE 位。DMP_ODR 不可动态改，需按停用-改-复位-重启流程。

> [!tip] 定位
> 对 1 kHz 姿态流水线而言 APEX 帮不上忙（它不做姿态融合，[DMP 融合是上一代 MPU 系的思路](../../知识/姿态解算/IMU姿态解算算法演进.md)）；它的价值在低功耗待机链路：Sleep(7.5 µA) + 加计 LP + WoM 唤醒 MCU。

## 10. 时钟体系与多 IMU 同步（CLKIN）

内部时钟两种来源（`INTF_CONFIG1.CLKSEL`）：RC 振荡器，或"PLL 优先、无则 RC"（默认 01，手册推荐——陀螺工作时用其 MEMS 振荡器锁 PLL）。精度：

| 时钟源 | 25 °C 初始偏差 | 全温漂移 |
|--------|---------------|----------|
| RC 振荡器（陀螺关） | ±3% | ±3% |
| PLL（陀螺开） | ±1% | ±2% |
| **CLKIN 外部时钟** | 随源，31–50 kHz | 随源（如 32.768 kHz ±50 ppm 晶振） |

**CLKIN 是这颗片子的王牌**（pin 9，需 `INTF_CONFIG5.PIN9_FUNCTION=10` + `INTF_CONFIG1.RTC_MODE=1`）。喂入 50 ppm 级 32.768 kHz 时钟后：

1. **ODR 精度**：RC 模式 ±8%（±80,000 ppm）→ **±50 ppm**；PLL 模式 ±10,000 ppm → ±50 ppm。
2. **系统级灵敏度误差**：陀螺灵敏度本质是"每个采样周期积的角"，时钟误差 1:1 变成刻度误差——CLKIN 直接消掉。
3. **多器件/主机同步**：所有 IMU 与主机共用一个时钟域，采样点与周期已知且对齐，**16 颗 IMU 不再各漂各的，免去持续重同步**——这正是多节点动捕最痛的点；手册点名 EIS、导航、机器人受益。

> [!note] 两个 CLKIN 连带效应（Use Notes §12.5/12.7）
> - ODR 按 f_CLKIN/32 缩放：32.768 kHz 输入时名义 500 Hz 实际是 **512 Hz**，1 kHz 档实际 1024 Hz——解算 dt 必须跟着改。
> - FIFO 时间戳间隔需要缩放：RTC_MODE=1 且 CLKIN 时乘 **32.768/30**（例：读数 915.5 µs × 32.768/30 = 1000 µs）；未用 CLKIN 时乘 **32/30**。不缩放直接积分会引入 ~6.7% 的系统性 dt 误差。
> - FSYNC 不用时（pin 9 作他用或悬置需求）应将 FSYNC 输入接 GND。

## 11. 关键寄存器速查

访问模型：`REG_BANK_SEL`（0x76，任意 Bank 可见）切换 Bank 0–4；芯片上电处于 Sleep。

| Bank | 内容 |
|------|------|
| 0 | 运行时全部：数据、FIFO、中断、电源、ODR/FSR、UI 滤波、APEX 开关 |
| 1 | 陀螺静态配置（AAF/Notch）、自检数据、时间戳读回、接口配置 4/5/6 |
| 2 | 加计静态配置（AAF）、自检数据 |
| 4 | APEX 参数、WoM 阈值、APEX 中断路由、用户偏置 OFFSET_USER |

Bank 0 高频寄存器：

| 地址 | 名称 | 要点 |
|------|------|------|
| 0x11 | DEVICE_CONFIG | bit4 SPI Mode0/3↔1/2；bit0 软复位（写 1 后等 1 ms） |
| 0x14 | INT_CONFIG | INT1/INT2 极性、推挽/开漏、脉冲/锁存 |
| 0x16 | FIFO_CONFIG | bit7:6 FIFO 模式（Bypass/Stream/Stop-on-Full） |
| 0x1D–0x2A | TEMP/ACCEL/GYRO_DATA | 传感器数据，默认大端 |
| 0x2D | INT_STATUS | DRDY/FIFO 水位/FIFO 满等，读清 |
| 0x2E/0x2F | FIFO_COUNTH/L | 读 L 锁存；单位受 FIFO_COUNT_REC 影响 |
| 0x30 | FIFO_DATA | FIFO 读端口 |
| 0x4B | SIGNAL_PATH_RESET | DMP_INIT/DMP 内存复位/FIFO_FLUSH/时间戳锁存 |
| 0x4C | INTF_CONFIG0 | 无效样本策略、FIFO 计数单位/端序、禁用 SPI/I²C |
| 0x4D | INTF_CONFIG1 | **CLKSEL、RTC_MODE**、加计 LP 时钟源 |
| 0x4E | **PWR_MGMT0** | GYRO_MODE/ACCEL_MODE/温度使能；切模式后 200 µs 勿写寄存器 |
| 0x4F | **GYRO_CONFIG0** | bit7:5 FSR（默认 ±2000dps），bit3:0 ODR（默认 1 kHz） |
| 0x50 | **ACCEL_CONFIG0** | bit7:5 FSR（默认 ±16g），bit3:0 ODR（默认 1 kHz） |
| 0x51 | GYRO_CONFIG1 | 温度 DLPF 带宽；陀螺 UI 滤波阶数 |
| 0x52 | GYRO_ACCEL_CONFIG0 | 高 4 位加计 / 低 4 位陀螺 UI 滤波带宽档 |
| 0x53 | ACCEL_CONFIG1 | 加计 UI 滤波阶数 |
| 0x54 | TMST_CONFIG | 时间戳使能/分辨率 1µs↔16µs/差分模式 |
| 0x56 | APEX_CONFIG0 | 各 APEX 功能使能 + DMP_ODR |
| 0x5F | **FIFO_CONFIG1** | HIRES/温度/陀螺/加计入 FIFO 使能、水位比较模式、续读 |
| 0x60/0x61 | FIFO_CONFIG2/3 | 水位 FIFO_WM[11:0]，禁 0 |
| 0x63/0x64 | INT_CONFIG0/1 | 中断清除方式；**INT_ASYNC_RESET 写 0**；≥4 kHz 改 8 µs 脉宽 |
| 0x65/0x66 | INT_SOURCE0/1 | INT1 路由：DRDY/FIFO/WoM/SMD |
| 0x70 | SELF_TEST_CONFIG | 六轴自检使能（响应 = 开自检输出 − 关自检输出） |
| 0x75 | WHO_AM_I | 0x47 |

Bank 1/2 静态配置：GYRO_CONFIG_STATIC2–10（0x0B–0x13，AAF 开关/DELT/DELTSQR/BITSHIFT + Notch COSWZ/带宽）；ACCEL_CONFIG_STATIC2–4（Bank2 0x03–0x05，加计 AAF）；INTF_CONFIG4（0x7A，3/4 线 SPI）、INTF_CONFIG5（0x7B，**PIN9_FUNCTION**: INT2/FSYNC/CLKIN）、INTF_CONFIG6（0x7C，I3C 使能组）。

典型初始化顺序（综合手册要求）：软复位 → 等 1 ms → 读 WHO_AM_I 校验 0x47 → `INT_ASYNC_RESET=0` → Bank1/2 配 AAF（如需）→ Bank0 配 FSR/ODR/UI 滤波 → 配 FIFO 与水位、中断路由 → `PWR_MGMT0` 开 LN 模式 → 等 200 µs（勿写寄存器）→ 陀螺稳定约 30 ms 后数据可信。

## 13. 封装与布局装配要点

| 参数 | 值 |
|------|-----|
| 封装 | LGA-14，NiAu 焊盘 |
| 尺寸 | 2.5 × 3.0 mm，厚 0.85–0.91–0.97 mm |
| 焊盘 | 引脚间距 0.5 mm，引脚宽 0.2–0.3 mm，共面度 ≤0.08 mm |
| 轴向 | 顶视：+X 向右、+Y 向上、+Z 出屏；Pin 1 圆点定位（Figure 15） |

> [!note] MEMS 装配要点
> - MEMS 结构晶圆级气密封装，抗 20,000 g 冲击，但**焊接与板弯应力会转化为零偏偏移**——器件级零偏指标（陀螺 ±0.5 dps、加计 ±20 mg）本就是"板级"条件下给出的。工程实践：远离螺丝孔与板边、避免拼版 V-CUT 应力路径、回流焊后再做零偏标定（标定值写 OFFSET_USER 或存 Flash）。
> - 具体回流曲线、PCB 设计与操作规范手册未展开，指向 InvenSense 应用笔记 **AN-IVS-0002A-00**（MEMS Handling）。
> - 多 IMU 板注意 CLKIN 走线的时钟完整性：31–50 kHz 频率低，串扰风险小，但要避免与 SPI 高速线长距离并行。
