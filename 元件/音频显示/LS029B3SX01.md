# LS029B3SX01

> Sharp 2.89 寸 CG-Silicon TFT LCD 微显示模组。1440×1440, MIPI DSI 双端口, 90Hz。SPEC LCY-1315802C Rev.C, 38 pages.

## 1. 身份与选型

- **型号**: LS029B3SX01
- **类型**: CG-Silicon (Continuous Grain Silicon) TFT LCD
- **分辨率**: 1440 × 1440 (1:1 方形)
- **尺寸**: 2.89 寸对角线 (73.4mm)
- **有效区域**: 51.84 × 51.84mm
- **色彩**: 16,777,216 色 (24-bit RGB)
- **接口**: MIPI DSI (双端口, 8 lane 无压缩 / 4 lane 压缩)
- **驱动 IC**: NT35597 (内置 1/3 RAM + NVM)
- **连接器**: JAE WP7B-P040VA1 (40-pin Board-to-Board)
- **厂商**: Sharp Corporation

> [!warning] 注意
> 项目页中曾标注为 "0.29 寸 Micro OLED, 1280×720"——实际为 **2.89 寸 CGS LCD, 1440×1440**。
> 0.29 寸是 2.89 寸小数点错位；LS029B3SX 系列没有 1280×720 分辨率，全部为方形。

## 7. 驱动初始化

### 驱动 IC

NT35597，内置 1/3 RAM 和 NVM（非易失存储器）。Sharp 出厂时已将大部分寄存器配置烧录进 NVM，因此初始化只需 5 条 DCS 命令。

> 对比：纯 NT35597 面板（如 Linux 内核 `panel-truly-nt35597.c`）需要 200+ 条命令；LS029B3SX01 的 NVM 自动加载省掉了这些。

### 硬件上电序列 (Table 17)

```
1. XRES = L
2. IOVDD 1.8V 上电 → 等稳定 (>0ms)
3. AVDD+ 5.5V 上电 → 等 >1ms (上升斜率 >0.2ms)
4. AVDD- -5.5V 上电 → 等 >10ms
5. XRES = H → ≥20µs → XRES = L → ≥20µs → XRES = H
6. 等 ≥10ms [NVM 自动加载完成, 面板进入 Sleep Mode]
```

### DCS 初始化命令 (LP 模式)

| 序号 | 命令 | Data Type | 说明 |
|------|------|-----------|------|
| 1 | `0xFF 0x10` | `0x15` (DCS Short Write 1p) | Page Select → CMD1 |
| 2 | `0x11` | `0x05` (DCS Short Write 0p) | Sleep Out |
| | | | **等 ≥100ms** |
| 3 | `0xFF 0x10` | `0x15` | Page Select → CMD1 |
| 4 | `0x29` | `0x05` | Display On |
| | | | **等 ≥40ms** |

> [!warning] 必须在 CMD1 页面（`0xFF 0x10`）下发送 Sleep Out 和 Display On。XRES 复位后默认进入 CMD1 页面，但 Sleep Out 后会自动退出；Display On 前必须再次切回 CMD1。

### TC358870 驱动配置

#### DCS 命令通过 LPTX 寄存器发送 (LP 模式, 视频启动前)

```c
// 每一条 DCS Short Write (1 param, DT=0x15):
LPTX_TYPE        (0x022C) = 0x00000384   // Trigger0 + LPDT=1 + ALL lanes
DSI_LPTX_PKT_HDR (0x0230) = 0x10FF0005   // DT=0x15, WC0=0xFF, WC1=0x10
DSI_LPTX_REQ     (0x0228) = 0x01         // 触发
while(!(read(0x0220) & 0x01));           // 等 LpTxDone
write(0x0220) = 0x01;                    // 清中断

// 每一条 DCS Short Write (no param, DT=0x05):
DSI_LPTX_PKT_HDR (0x0230) = 0x00110005   // DT=0x05, WC0=0x11 (Sleep Out)
// 或
DSI_LPTX_PKT_HDR (0x0230) = 0x00290005   // DT=0x05, WC0=0x29 (Display On)
```

#### DSI PLL (40MHz REFCLK, 目标 400MHz, 800Mbps/lane)

```
MIPI_PLL_CONF (0x02AC) = 0x0003793F   // 4字节: {0x3F,0x79,0x03,0x00}
//   bit[17:16] LBW = 11 (max bandwidth)
//   bit[15:12] PRD = 7 → ÷8
//   bit[11:10] FRS = 2 → ÷4
//   bit[8:0]   FBD = 319 → ×320
//   pll_clk = 40MHz × 320/8 / 4 = 400MHz → 800Mbps/lane
MIPI_PLL_CTRL (0x02A0) = 0x03          // MP_ENABLE + MP_CKEN
while(!(read(0x02A8) & 0x01));          // 等 PLL lock

PPI_DPHY_POWERCNTRL (0x0284) = 0x1F     // 全部 Lane 上电
DSITX_CLKEN   (0x0108) = 0x01
```

#### DSI 模式配置

```
MODE_CONFIG (0x0110) = 0x16    // IndMode=1(LP命令), HSYNC/VSYNC=ActiveLow(DSI要求)
FUNC_MODE   (0x0150) = 0x00    // HsCkMd=0(非连续时钟), EoTpEn=0
DSITX_MODE  (0x017C) = 0x00    // BlankPkt_En=0(LP消隐), Pulse Mode
LANE_ENABLE (0x0118) = 0x1F    // CK + D0-D3 全部使能
```

#### DSI 视频时序 (对应 1694×1712 总量, DSI0 半幅)

```
DSI_HSYNC_WIDTH (0x018C) = 0x0004   // HSA ≈ 4 bytes
DSI_HBPR       (0x0190) = 0x0092   // HBP ≈ 146 bytes (150-4)
// TC358870 自动从 HDMI 输入时序继承, 消隐编码由 DSITX_MODE 控制
```

### MIPI 双端口模式

- **Port A** (DSIA): 4 data lanes + 1 clock lane → 接 TC358870 DSI0
- **Port B** (DSIB): 4 data lanes + 1 clock lane → 接 TC358870 DSI1
- **EN1PORT** 引脚 = L (双端口使能)
- 单端口模式下仅 DSIA 工作，DSIB 可悬空

### CG-Silicon 特点

- 相比 a-Si TFT，电子迁移率更高 → 驱动 IC 可部分集成到玻璃上
- 支持 DC Driving / 1H Z-Inversion，降低闪烁
- Normally Black 模式，黑色更纯净

## 8. 封装

- **外形尺寸**: 54.24 × 59.02 × 1.365mm (不含 FPC 突出部)
- **重量**: 7g
- **连接方式**: 40-pin Board-to-Board Plug (JAE WP7B-P040VA1)
- **FPC**: 含 LED 背光供电、MIPI 信号、控制信号
- **面板结构**: LCD 面板 + COG 驱动 + FPC + LED + 棱镜片 + 扩散片 + 导光板 + 反射片 + 塑料框架
