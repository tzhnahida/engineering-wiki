# TC358870
> 东芝 (Toshiba) HDMI 1.4b → 双 MIPI DSI 视频桥接芯片。4K×2K@30fps。VFBGA80。
> 不含缩放器（Scaller-less），是 TC358779 的简化后继——仅做格式/分割转换，不做分辨率缩放与去隔行。

## 6. 核心功能

[📷 _llm/raw/assets/datasheets/tc358870/tc358870_p7_fig1.jpg|640]
*系统概览：HDMI-Rx → 分割/格式转换 → 双 MIPI DSI-Tx*

- **HDMI → MIPI 桥接**：将 HDMI 1.4b 视频流转换为双 MIPI DSI（支持双端口模式扩展带宽）→ [视频显示/MIPI DSI](../../知识/视频显示/MIPI DSI.md)
- **4K 支持**：输入 4K×2K@30fps，通过双路 DSI 输出保证 8 Gbps 带宽足够传输
- **音频提取**：从 HDMI 流中提取音频，通过 I2S 接口输出 5.1/7.1ch 环绕声 → 数字音频接口 I2S
- **HDCP 1.4 解密**：支持内容保护解密，用于合法播放受保护内容
- **EDID 管理**：片上 1KB EDID SRAM，可通过 I2C 在线更新（见第 8 节）
- **颜色空间转换**：YCbCr 4:2:2 ↔ 4:4:4 / RGB ↔ YCbCr，无缩放/去隔行
- **CEC 遥控**：HDMI CEC 总线控制，支持 NEC 红外远程输入

## 8. EDID 方案：外挂 24C02 EEPROM

> [!warning] 公开手册未写明内部 EDID_SRAM 的 I2C 子地址。
> 因此最稳妥的方案是外挂一颗标准 I2C EEPROM (AT24C02)。

### EDID_MODE 寄存器 (0x85E0)

| bit[1:0] | 模式 | 行为 |
|----------|------|------|
| `00` | **DDC 直通外部 EEPROM** | 芯片完全不响应 DDC 读取，DDC_SCL/DDC_SDA 直通外部总线 |
| `01` | 内部 EDID-RAM + DDC2B | 芯片响应 DDC 读取（内部 RAM） |
| `1x` | 内部 EDID-RAM + E-DDC | 芯片响应 DDC 读取，支持 E-DDC segment pointer |

**模式 `00` = 外挂 EEPROM**。DDC 总线上的 EDID 读请求 (I2C 地址 0xA0) 由外部 EEPROM 应答。

### 电路连接

```
HDMI +5V ──┬──────────────────── HPDI (A4)
           ├── 47kΩ ──────────── DDC_SCL (A3) ─── AT24C02 SCL
           ├── 47kΩ ──────────── DDC_SDA (B3) ─── AT24C02 SDA
           │                                       AT24C02 VCC ← +5V
           │                                       AT24C02 WP → GND
 HPD ──────┤
           ├── 100kΩ ─────────── GND
           └── 1kΩ ──────────── HPDO (B4)

 REXT (A1) ─── 2kΩ±1% ──────── VDD33_HDMI
 INT (J3) ─── 10kΩ → VDDIO18 或 GND  (上拉=0x1F, 下拉=0x0F)
```

### AT24C02 和调试注意事项

- SOP-8 封装，与 24C256/512 脚位兼容
- I2C 地址固定 0x50（A0/A1/A2 全接地）
- 供电用 HDMI +5V（不用系统电源）→ 电平匹配 DDC 5V 总线
- FT（5V 容忍）MCU（STM32F103/G431）可直接挂 DDC 总线调试写 EDID
- I2C 开漏：MCU 只拉低不推高，高电平由外部 47kΩ 上拉提供

### 9.3 MIPI_PLL_CONF `0x02AC` (32-bit)

```
bit[17:16] MP_LBW     Loop BW: 00=25% 01=33% 10=50% 11=Max
bit[15:12] MP_PRD     输入分频 = PRD+1
bit[11:10] MP_FRS     后分频 2^FRS
bit[9]     MP_LFBREN  下限移除使能
bit[8:0]   MP_FBD     反馈分频 = FBD+1

dsitx_pll_clk = REFCLK × (FBD+1)/(PRD+1) × 1/2^FRS

800Mbps: FBD=319, PRD=7, FRS=2, LBW=11
→ 0x02AC={0x3F,0x79,0x03,0x00}
```

### 9.10 初始化序列 (诱骗器模式)

```
1. SYSCTL     (0x0002) = 0x3F01   // 全复位+SLEEP
2. SYSCTL     (0x0002) = 0x0000   // 释放
3. ConfCtl0   (0x0004) = 0x0004   // AutoIndex=1
4. ConfCtl1   (0x0006) = 0x0008   // dcs_clks=1
5. SYS_FREQ0  (0x8540) → REFCLK/10000 LSB
6. SYS_FREQ1  (0x8541) → REFCLK/10000 MSB
7. EDID_MODE  (0x85E0) = 0x00     // 外挂EEPROM模式
8. DDCIO_CTL  (0x84F4): 0→1      // DDCIO 上电
9. DDC_CTL    (0x8543) = 0x04     // DDC_ACTION=1
10. PHY_ENB   (0x8413) |= 0x01    // 退出 Suspend
11. PHY_RSTX  (0x8414) |= 0x01    // 退出复位
12. VOUT_FMT  (0x8A00) = 0x00     // RGB
13. VOUT_CSC  (0x8A08) = 0x00     // 无转换
14. ConfCtl0  (0x0004) = 0x0007   // Vtx0_en+Vtx1_en+AutoIndex
15. INIT_END  (0x854A) = 0x01     // 初始化完成
16. HPD_CTL   (0x8544) = 0x01     // HPDO=H, 通知源端
```

## 11. 视频数据通路详解

TC358870 不含缩放器，因此视频数据通路是**直通 + 格式转换 + 分割**的流水线：

```
HDMI TMDS-Rx → HDCP 解密 → 视频 FIFO → 颜色空间转换 → Splitter → DSI0-Tx
                                                                   → DSI1-Tx
```

### 11.1 HDMI 输入能力核算

| 项目 | 数值 |
|------|------|
| TMDS 时钟上限 | 297 MHz |
| 最大像素率 | 297 Mpixels/s |
| 输入视频带宽 | 7.2 Gbps（≈ 297 Mpix/s × 24 bpp = 7.128 Gbps） |
| 4K@30 有效数据 | 3840 × 2160 × 30 × 24 ≈ 5.97 Gbps（含消隐后即 7.128 Gbps，恰为上限） |

### 11.2 双 DSI 分割输出

芯片内置 Splitter 模块（寄存器 0x5000–0x5FFF），将 HDMI 输入行按像素水平切分为左右两半：

- **默认**：DSI0 输出左半、DSI1 输出右半
- **可通过 STX0_FPX / STX0_LPX 精确控制**各通道的起始和结束像素（0–4095）
- 单段最大宽度 **2048 像素** + 允许重叠（两路数据长度可以不同）
- 两路 DSI 的 Hsync 偏移控制在 **10 ByteClk** 以内
- 每 DSI 输出：4 lane × 最高 1 Gbps/lane，双路合计 8 Gbps

> [!tip] 带宽余量
> 4K@30 含消隐的输入数据率为 7.128 Gbps，双 DSI 满速 8 Gbps（8 lane × 1 Gbps），留有约 12% 余量——这正是"必须双路 DSI"的原因：单路 4 Gbps 不足以承载 4K@30。单链路模式下最大水平宽度 2558 像素（24-bpp）/ 3411 像素（16-bpp）。

### 11.3 颜色格式与转换

| 输入格式 | 输出格式 | 转换方式 |
|----------|----------|----------|
| RGB 888（24-bpp） | RGB 888 / RGB 666 | 直通或降位 |
| YCbCr 4:4:4（24-bpp） | RGB 888/666, YCbCr 4:4:4 | CSC 引擎 |
| YCbCr 4:2:2（24-bpp） | RGB 888/666, YCbCr 4:4:4 | 4:2:2 → 4:4:4 插值 |
| RGB 888 | YCbCr 4:4:4 / 4:2:2 | CSC 引擎 |

> [!note] 色彩深度限制
> HDMI 输入**不支持 Deep Color**（30/36-bit），最高 24-bpp。DSI 输出可选择 RGB666 或 RGB888，RGB666 时低位补零：R={R[5:0], 2'b00}。

### 11.4 彩条发生器

地址 `0x7000–0x709F` 的内置彩条（Color Bar）模式可用于**无源调试**——不需要 HDMI 输入即可让 DSI 输出已知图案，验证面板连接和 DSI 时序是否正常。

## 12. HDMI-Rx 接收特性

### 12.1 TMDS 接收器与自动终端匹配

[📷 _llm/raw/assets/datasheets/tc358870/tc358870_p16_fig1.jpg|480]

HDMI-Rx PHY 内建 **自动终端电阻调节**（Automatic Adjustment of Terminus Resistance），通过 REXT 引脚的外接精密电阻作为基准：

- **REXT (A1)** 必须外接 **2 kΩ ±1%** 到 VDD33_HDMI
- PHY 根据此参考自动校准内部终端阻抗，使差分阻抗匹配 HDMI 电缆的 100 Ω 特性阻抗
- 电阻精度直接影响 TMDS 信号完整性——务必使用 ±1% 或更高精度

TMDS 信号检测链路（从 SYS_STATUS 寄存器读取）：

```
S_DDC5V (bit 0)  →  5V 检测
  → S_TMDS (bit 1)  →  TMDS 时钟检测
    → S_PHY_PLL (bit 2)  →  PHY PLL 锁定
      → S_PHY_SCDT (bit 3)  →  信号锁定（TMDS 数据同步）
        → S_SYNC (bit 7)  →  帧/行同步检测
```

> [!tip] 调试 HDMI 无输出时，按此链路逐级检查 SYS_STATUS 位，定位中断点。

### 12.2 5V 检测与 HPD 控制

- **HPDI (A4)**：检测 HDMI 源端提供的 +5V 电源。防抖时间可通过 DDC_CTL[1:0] 配置（0/2/10/100 ms）
- **HPDO (B4)**：Hot Plug Detect 输出，开漏输出，通知源端芯片已准备好通信
  - **自动模式**（HPD_CTL[1]=1）：初始化完成后自动拉高
  - **手动模式**（HPD_CTL[1]=0）：通过 HPD_OUT0 (bit0) 软件控制

### 12.3 HDCP 1.4 解密

- 支持 HDCP 1.4 解密（需授权许可，非所有版本皆包含）
- 片上 eFuse 存储 HDCP 密钥
- 内部包含 HDCP 认证引擎（Authentication Engine）

### 12.4 DDC 接口

- DDC_SCL / DDC_SDA 为 5V 容忍、开漏，配上拉至 +5V（HDMI 源端供电）
- 电平参数：VIH ≥ 3.1V，VIL ≤ 1.7V

## 13. 音频通路

HDMI 音频提取路径：

```
HDMI 流 → 音频解包（Audio De-Packet）→ 音频 FIFO → I2S/TDM 输出
                           ↕
                     Audio PLL（追踪 ACR N/CTS）
```

### 13.1 音频输出接口

| 引脚 | Ball | 功能 | I/O |
|------|------|------|-----|
| A_SCK | K7 | I2S/TDM 位时钟（Bit Clock） | O |
| A_WFS | K5 | I2S 字时钟 / TDM 帧同步 | O |
| A_SD0 | J9 | I2S 数据线 0 / TDM 数据 | O |
| A_SD1 | J8 | I2S 数据线 1 | O |
| A_SD2 | J6 | I2S 数据线 2 | O |
| A_SD3 | J5 | I2S 数据线 3 | O |
| A_OSCK | J4 | 音频过采样时钟（256fs） | O |

### 13.2 音频能力

| 参数 | 规格 |
|------|------|
| 通道数 | 最多 8 通道（5.1 / 7.1） |
| 采样率 | 最高 192 kHz |
| 位深 | 16 / 18 / 20 / 24-bit（取决于 HDMI 输入） |
| 时隙宽度 | 32-bit 固定 |
| 接口模式 | I2S（左/右对齐）或 TDM 8 通道 |
| HBR 音频 | 带宽 >12 MHz 时自动分散到 4 路 I2S |
| 主时钟 | 仅 Master 模式，输出 A_OSCK = 256 × fs |

### 13.3 音频 PLL 与外部 LPF

[📷 _llm/raw/assets/datasheets/tc358870/tc358870_p16_fig3.jpg|560]

音频 PLL 通过追踪 HDMI ACR（Audio Clock Recovery）包中的 N/CTS 值来恢复音频时钟，恢复路径为：

```
DAOUT (J2) → 外部 LPF → PCKIN (K1) → 片内 PLL → 音频时钟
```

外部 LPF 由两级滤波构成：

| 级 | 元件 | 作用 |
|----|------|------|
| 第一级 LPF | 1.5 kΩ + 5 pF（串联） | DAOUT 输出滤波，去除高频噪声 |
| 第二级 LPF | 1.0 kΩ + 1000 pF（RC 并联到地） | PLL 相位比较器输出的低通滤波 |
| 缓冲晶体管 | 2SC2712Y | 隔离 DAOUT 与 LPF，提供驱动能力 |
| 偏置电容 | 0.1 μF / BIASDA 各接 0.1 μF 到 AVSS | PLL 偏置滤波 |
| AVSS 耦合 | 各 0.1 μF 到地 | 模拟地退耦 |

> [!warning] 音频时钟 LPF 是关键模拟电路
> DAOUT→PCKIN 之间的外部 LPF 参数**必须严格按手册取值**。更改元件值可能导致 Audio PLL 失锁、音频时钟抖动或无声。不使用时，DAOUT 悬空，BIASDA、PCKIN、PFIL 各接 0.1 μF 到 AVSS。

## 14. 时钟体系

TC358870 内部有三个时钟域：

```
┌─────────────────────────────────────────────────────┐
│  REFCLK (40–50 MHz)                                 │
│    ├── SYS_FREQ ÷10000 → 系统时钟参考                  │
│    ├── MIPI PLL ×N → DSI 位时钟 (≤1 Gbps/lane)       │
│    └── 音频 PLL (外部 LPF 反馈) → I2S 时钟             │
│                                                       │
│  HDMI TMDS 时钟 (≤297 MHz)                            │
│    └── N/CTS → 音频 PLL 参考 → 音频时钟恢复             │
└─────────────────────────────────────────────────────┘
```

### 14.1 REFCLK 要求

| 参数 | 条件 | 规格 |
|------|------|------|
| REFCLK 频率 | 输入 | **40–50 MHz** |
| SYS_FREQ 寄存器 | REFCLK(Hz) / 10000 | 40 MHz → 0x0FA0, 50 MHz → 0x1388（默认） |
| REFCLK 稳定性 | 上电序列 | 必须在 INIT_END 前稳定 |

### 14.2 MIPI DSI PLL

DSI 位时钟由 MIPI PLL 从 REFCLK 倍频产生。配置寄存器 `MIPI_PLL_CONF` (0x02AC)：

```
dsitx_pll_clk = REFCLK × (FBD+1) / (PRD+1) / 2^FRS

800 Mbps/lane 示例（REFCLK = 40 MHz）：FBD=319, PRD=7, FRS=2
40 MHz × (319+1) / (7+1) / 2^2 = 40 × 320 / 8 / 4 = 400 MHz
DDR 双沿采样 → 有效数据率 = 400 MHz × 2 = 800 Mbps ✓
```

> [!tip] MIPI PLL 环路带宽可选 25%/33%/50%/Max（MP_LBW 位），高带宽锁定快但抖动略大，建议从 33% 或 50% 开始调试。

### 14.3 Audio PLL

独立于 MIPI PLL 工作，通过 `APLL_CTL (0x84F0)[0] = APPL_ON` 使能。PLL 参考来自 DAOUT→外部 LPF→PCKIN 反馈环，N/CTS 值由 HDMI ACR 包解析得到。

## 15. 电源与时序要求

### 15.1 电源域与建议上电顺序

| 电源域 | 典型值 | 供电对象 |
|--------|--------|----------|
| VDDC11 | 1.15 V | 数字核心（与 VDD11_HDMI 可同源） |
| VDD11_HDMI | 1.15 V | HDMI PHY 数字部分 |
| VDD12_MIPI0/1 | 1.2 V | DSI PHY（两路独立供电） |
| VDDIO18 | 1.8 V | 系统 IO、音频、IR、I2C、复位 |
| VDDIO33 | 3.3 V | DDC、CEC、HPD、部分 IO |
| VDD33_HDMI | 3.3 V | HDMI PHY 模拟部分、Audio PLL |

建议上电顺序：**VDDC11 / VDD11_HDMI → VDDIO18 → VDDIO33 / VDD33_HDMI → VDD12_MIPI0/1**。RESETN 保持低电平直到所有电源稳定。

### 15.2 复位与初始化时序

1. **RESETN (K8)** 拉低至少 **1 ms**（所有电源稳定后）
2. 释放 RESETN → INT 引脚自动从"地址采样"切换为"中断输出"
3. REFCLK 必须在初始化前稳定
4. 软件初始化序列见第 9.10 节

> [!note] INT 引脚的双重角色
> RESETN 有效期间，INT 输入电平被锁存：上拉选择 I2C 地址 0x1F，下拉选择 0x0F。释放后自动切换为中断输出。

### 15.3 TMDS 反向电流保护

> [!danger] TMDS ESD 反向电流
> HDMI TMDS 输入引脚带有电源轨间的 ESD 保护二极管。当系统断电而 HDMI 源端仍供电时，TMDS 信号线上的电流会通过 ESD 二极管倒灌入 VDD33_HDMI，进而损坏其他 3.3V 电源轨。

手册提供三种防护方案：

| 方案 | 描述 | 适用场景 |
|------|------|----------|
| 外部开关（Case 1） | 在 TMDS 差分线上加反流开关 | 常规设计 |
| 带反流保护的 LDO（Case 2） | 选用有反向电流保护功能的 3.3V LDO 为 VDD33_HDMI 供电 | 简化设计 |
| VDDIO18=3.3V 共轨（Case 3） | 将 VDDIO18 也用 3.3V 供电，VDD33_HDMI 通过外部开关隔离 | I/O 电平为 3.3V 的系统 |

所有 VSS 应通过**铁氧体磁珠（Ferrite Bead）** 与电源地 AC 隔离，以抑制 EMI。
