# LT7911D
> 龙讯 (Lontium) 视频接口桥接芯片。Type-C (DP Alt-Mode) / DP 1.2 / eDP 1.4 转单/双端口 MIPI DSI/CSI，带音频输出。QFN64。

### 4.4 上电时序与复位（手册 §8.6，Figure 8.6.1）

1. **VCC33 先上电**（有效阈值 2.97V）
2. **VDD (1.2V) 至少晚 1ms 上电**（有效阈值 1.08V）——内部复位信号在 1.2V 就绪后才释放
3. **外部 RESET# (RSTN) 至少再晚 1ms 释放**（判决电平 0.8V）；RSTN 低电平有效，Schmitt 输入

> [!note] 手册未规定下电顺序与 RSTN 最短脉宽（—）。

## 5. 通信/接口特性

### 5.1 Type-C / USB PD 前端

- 符合 VESA DisplayPort Alt Mode on USB Type-C 标准 v1.0；兼容 USB Type-C V1.1
- 符合 USB Power Delivery Rev 2.0
- 符合 HDMI 1.4b Alt Mode on USB Type-C 规范 V1.0
- **内置 CC 控制器**：插入检测与正反插方向检测由芯片自行完成，无需外置 CC 逻辑芯片
- **双路 CC（UCC1/UCC2）**：一路用于充电器、一路用于正常通信——支持边充电边传输
- 供电模式可选：从手机端取电或适配器供电（Power from phone or adapter mode selection）

### 5.2 DP/eDP 输入

- DP 1.2：1.62 / 2.7 / 5.4 Gbps（RBR/HBR/HBR2），1/2/4 lane 可配置，支持 lane swap
- eDP 1.4：支持 eDP 认证——ASSR（Alternative Scramble Seed Reset）与 Alternative Framing
- 链路训练：Full Link Training + **Fast Link Training**（嵌入式 eDP 系统使用快速训练可节省系统时间 ≥400ms）
- **自适应接收均衡**：补偿 PCB / 线缆 / 连接器损耗，适合长线应用
- HDCP 1.3（DP 输入侧内容保护）
- 固件升级通道：**AUX 或 IIC**
- HPD：RX_HPD（pin 23）开漏输出热插拔信号
- 输入锁定后 MCU 经 MSA 寄存器读取恢复的时序参数以匹配 ASSR；调试 Full Link Training 时 DPCD 寄存器可经系统 I2C 访问

### 5.3 MIPI DSI/CSI 输出

- 兼容 DCS 1.02、D-PHY 1.2、DSI 1.02、CSI-2 1.0（DSI/CSI 可配置）
- **1/2 端口可配置**；每端口 1 clock lane + 1~4 data lane 可配置
- 速率 80 Mb/s ~ 1.5 Gb/s / data lane（时钟通道最高 750MHz）；双端口 8 lane 总带宽 12 Gbps
- Data lane 交换、极性交换、端口交换（port swap）
- 支持 Non-burst 与 Burst 视频模式
- 双端口左右拼接：每半屏最大 **64 像素重叠**（overlap）
- **Stream Copy**：同一视频流复制到两个端口（双面板同显）
- **Side-by-side 3D**：左眼数据发一个端口、右眼数据发另一个端口
- 像素格式：RGB888 / RGB666 / Loosely RGB666 / RGB565 / 16/20/24-bit YCbCr 4:2:2 / 12-bit YCbCr 4:2:0

### 5.4 音频与控制

- 音频输出：SPDIF 或 2 声道 IIS（MCLK/SCLK/WS/D0）
- 控制接口：**I2C 从机**（CSDA/CSCL），支持 100kHz / 400kHz——手册未提 I2C 主机模式（旧表"主机模式"无出处，已删）
- 内置 CSC：YCbCr 4:4:4 ↔ RGB、YCbCr 4:2:2 ↔ YCbCr 4:4:4
- 内嵌 EDID shadow（EDID 缓冲） → [视频显示/HDMI EDID](../../知识/视频显示/HDMI%20EDID.md)

## 6. 核心功能

[📷 _llm/raw/assets/datasheets/lt7911d/lt7911d_p8_fig1.jpg|600]
*Figure 7.1.1 — 功能方框图：Type-C DP Alt-Mode → MIPI DSI/CSI*

- **视频桥接**：DP 1.2 / eDP 1.4（含 Type-C DP Alt-Mode）→ MIPI DSI 或 CSI-2 输出 → [视频显示/MIPI DSI](../../知识/视频显示/MIPI%20DSI.md)
- **高带宽**：输入 4 lane @ 5.4Gbps（21.6Gbps）+ 自适应均衡；输出双端口 4+4 lane @ 1.5Gbps（12Gbps）
- **双面板 / 3D 分发**：2D 同一视频流可映射到两块独立面板；3D 格式左半数据发一个面板、右半发另一个面板
- **内置 MCU + 嵌入式 Flash**：EDID 缓冲（EDID shadow）、DP/eDP 输入检测、自动进入省电模式；MSA/DPCD 可见性与 Fast Link Training 细节见 §5.2
- **HDCP 1.3**：DP 输入侧解密（手册未提输出侧重新加密）
- **内置 CSC**：YCbCr 4:4:4 ↔ RGB、YCbCr 4:2:2 ↔ 4:4:4 色彩空间转换
- **音频提取**：输出 SPDIF 或 2 声道 I2S

### 7.5 布线与去耦要求（手册 §6.1）

- 所有差分对按 **100Ω ±10%** 差分阻抗布线；对内长度失配 **<5mil**，总走线尽量短
- 差分对建议走顶层或底层，信号路径**不打过孔**
- 每个电源脚就近放置至少 **0.1μF + 0.01μF** 去耦电容；去耦电容到电源脚走线最短，避免大电流环路与走线电感

**典型连线**：Type-C (DP Alt-Mode) / DP / eDP 源 → LT7911D → 单/双 MIPI DSI 面板；或 MIPI CSI-2 → SoC 摄像头接口（VR / 手机方案）

## 9. 寄存器与 I2C 编程

> [!warning] 完整寄存器手册为 Lontium 保密文档(NDA)
> 以下信息来自开源驱动逆向和同类芯片推理。**LT7911D 具体的寄存器地址可能与 EXC 不同**，但架构一致。

### 9.1 I2C 寻址与页映射

LT7911D 的 I2C 从机地址尚未在开源代码中确认。同厂 Lontium 芯片参考：
- **LT7911EXC**: `0x41` (7-bit)
- **LT9611**: 通常 `0x29` / `0x2b`
- **LT9211**: `0x2d` 或 `0x2b`

LT7911D 的寄存器使用**页映射 (paging) 机制**：
- 页选择寄存器: `0xFF` → 写入页码后，后续 I2C 读写映射到该页的 256 字节窗口
- 地址范围: `0x0000` ~ `0xE8FF` (最多 233 页)

### 9.3 MIPI DSI/CSI 输出配置 (综合 LT9211/LT9611/LT7911EXC)

LT7911D 的 MIPI 输出支持双端口 (Port 0 / Port 1)，每端口 1 Clock + 1~4 Data Lane，最高 1.5Gbps/lane。
以下是同厂 Lontium 芯片的 MIPI 寄存器关键地址，LT7911D 大概率使用相同或临近地址。

#### 9.3.1 MIPI Lane 配置

| 寄存器 | 页 | 功能 | 来源 |
|--------|-----|------|------|
| `0xD000` | D0 | **DSI Lane 数量**：`0x00`=4lane, `0x01`=1, `0x02`=2, `0x03`=3 | LT9211 |
| `0xD002` | D0 | DSI PHY 初始化值 (`0x05`) | LT9211 |
| `0x8300` | 83 | DSI Lane 数量 (双端口模式下 PortA) | LT9611 |

#### 9.3.2 MIPI PHY 配置 (RX Analog)

| 寄存器 | 功能 | 值 | 来源 |
|--------|------|-----|------|

#### 9.3.6 双端口模式配置

LT7911D 支持将同一视频流复制到两个 MIPI 端口 (Stream Copy Mode)，或将 3D 视频的左右眼分别发送到两个端口 (Side-by-Side)。

| 寄存器 | BIT | 功能 | 来源 |
|--------|-----|------|------|
| `0x8303` | 6 | 端口交换：PortB 作为主端口 | LT9611 |
| `0x8250` | 3:2 | Byte Clock 源选择 (`01`=PortB) | LT9611 |
| `0x830D` | — | 双端口模式 V Total 配置 | LT9611 |

#### 9.3.7 MIPI 输出状态检测

| 寄存器 | 功能 | 来源 |
|--------|------|------|
| `0xD082` | RX 检测到的 DSI 宽度 | LT9211 |
| `0xD083` | RX 检测到的 DSI 高度 | LT9211 |
| `0xD087` | PCR 锁定状态 (BIT3=1 锁定) | LT9211 |
| `0xE081-E083` | 固件版本 (24-bit) | EXC |

> [!note] LT7911D vs LT7911EXC MIPI 架构差异
> - **LT7911EXC**: eDP→DSI，所有 DSI 包生成和面板初始化由内部固件处理，驱动不直接配置 MIPI 时序
> - **LT7911D**: DP/eDP→DSI/CSI **双端口**，支持 stream copy 和 side-by-side 3D 模式。MIPI 配置更复杂，包含双端口协调逻辑

### 9.5 MSA 寄存器与 DPCD 寄存器

Datasheet R1.2 明确提到：
- **MSA 寄存器 (Main Stream Attribute)**：MCU 读取恢复的时序参数以匹配 ASSR 自动省电模式。这些寄存器通常映射到 DP 接收端地址空间，包含：
  - MSA 水平/垂直总像素数
  - MSA 水平/垂直有效像素数
  - MSA 像素时钟 (Mvid/Nvid)
  - MSA 颜色格式与位深
- **DPCD 寄存器 (DisplayPort Configuration Data)**：可通过系统 I2C 访问，用于调试链路训练：
  - DPCD 接收端能力字段 (0x00000～0x000FF)
  - Link 状态字段 (0x00200～0x0021F)
  - Sink 状态字段

### 9.6 固件更新流程 (参考 LT7911EXC)

```
1. 复位芯片 (RSTN: High→Low→High→Low, 各~20ms)
2. MCU Halt:  写 0xE0EE = 0x01
3. Flash 擦除: 写 0xE054=0x01, 0xE055=0x06, 触发擦除
4. 按页写入 (32 字节/页): 写地址到 0xE05A-C, 数据到 0xE05D
5. CRC 校验: 写 0xE07B 触发, 从 0x22 读取结果
6. MCU Run:   写 0xE0EE = 0x00
7. 读取版本: 读 0xE081～0xE083
```

## 10. 面板兼容性：带宽分析与替代方案

### 10.1 LT7911D 的能力上限

| 分辨率 | 刷新率 | 有效带宽 | +15%消隐 | MIPI 12G? | DP 17.3G? | 结论 |
|--------|--------|---------|----------|:---:|:---:|------|
| 1440×1440 | 90Hz | 4.5G | 5.1G | ✅ | ✅ | ✅ 可驱动 |
| 2880×2880 | 60Hz | 11.9G | 13.7G | ❌ 12G不够 | ✅ | ❌ MIPI瓶颈 |
| 2880×2880 | 90Hz | 17.9G | 20.6G | ❌ | ❌ | ❌ 两端都不够 |
| 3488×3488 | 120Hz | 35.0G | 40.3G | ❌ | ❌ | ❌ 严重不够 |

**根因：LT7911D 的 D-PHY 1.2 (1.5Gbps/lane, 12Gbps 总输出)，极限约 1440×1440@90Hz。**

### 10.2 D-PHY vs C-PHY

BOE VS032B3M (= 3488×3488 面板) 使用 **MIPI C-PHY** (2ch, 3 data lanes)，三态编码 2.28 bits/sym。LT7911D 的 D-PHY，**物理层不兼容**，无法互换。

### 10.3 3488×3488@120Hz 带宽需求

```
有效像素:  3488 × 3488 × 120Hz × 24bpp  = 35.0 Gbps
+15%消隐:                                = 40.3 Gbps
DSC 3:1 压缩后:                           = 13.4 Gbps ✅ 可行
```

### 10.4 替代桥接芯片 (HDMI/DP → MIPI C-PHY)

| 芯片 | 输入 | MIPI C-PHY | DSC | 封装 | 结论 |
|------|------|-----------|-----|------|------|
| **ITE IT6626** | HDMI 2.1 (48G FRL) | 双端口, 3 trio/port, **5.7 Gsym/s** | 透传 ✅ | QFN-76 | ⭐首选: HDMI 2.1 + C-PHY 5.7G |
| **Lontium LT7911UXE** | HDMI 2.1/DP 1.4a/Type-C | 四端口, **5.7 Gsym/s** | 编解码 ✅ | BGA-169 | ⭐⭐高配: 多输入+内置DSC |
| **Lontium LT7911UXD** | HDMI 2.1/DP 1.4a/Type-C | 双端口, 2.5 Gsym/s | — | QFN-88 | ✅ 小封装: C-PHY 1.0 |
| **ITE IT6625** | HDMI 2.0b (18G) | 双端口, 5.7 Gsym/s | — | QFN-76 | ⚠️ 输入仅18G,需GPU做DSC |
| ~~LT7911D~~ | DP 1.2/eDP 1.4 | ❌ D-PHY only | — | QFN-64 | ❌ 无C-PHY, 带宽不足 |

### 10.5 推荐数据路径

```
GPU (DSC编码) → HDMI 2.1 FRL (48Gbps, DSC流)
  → IT6626 (DSC透传)
    → MIPI C-PHY 双端口 (DSC压缩, 13.4Gbps)
      → R63455驱动IC (DSC解码)
        → 3488×3488 120Hz 面板
```

### 10.6 关键设计考量

- **DSC 必须在 GPU 端编码** —— IT6626 仅透传 DSC，不解码不重编
- **C-PHY 6 trio 总带宽 78 Gbps** —— DSC 压缩后仅需 13.4 Gbps，余量巨大
- **HDMI 2.1 FRL** 是关键 —— TMDS (18G) 无法承载 40 Gbps 未压缩流
- **R63455** 内置 DSC 解码器，与 IT6626 的 DSC 透传配对完美
- 如需 **DP 输入 + 片上 DSC 编码**，选 LT7911UXE
