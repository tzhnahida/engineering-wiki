# LLM Wiki 方法论

用 LLM 智能体构建持久化、可累积的知识库的方法论。

## 核心洞见

不同于传统 RAG 在查询时从原始文档检索，LLM **增量构建并维护一个持久化 Wiki**——位于用户与原始资料之间的结构化、互联的 Markdown 文件集合。知识编译一次、持续更新，而不是每次查询都重新推导。

## 三层架构

1. **原始资料** (`_llm/raw/`) —— 不可变的源文件。LLM 只读不写。
2. **知识库** (`元件/ 知识/ 来源/ 项目/`) —— LLM 生成的页面：元件页、概念页、来源摘要、项目页。LLM 全权维护。
3. **模式文档** (`CLAUDE.md`) —— 定义目录结构、命名规范和操作流程。与 LLM 共同演化。

## 三种操作

| 操作 | 说明 |
|------|------|
| **导入 (Ingest)** | 读取新资料，讨论要点，写摘要，更新相关页面，更新主页和日志 |
| **查询 (Query)** | 从主页定位相关页面，综合答案并附引用，有价值的内容归档为知识页 |
| **巡检 (Lint)** | 健康检查：矛盾、过时声明、孤立页面、缺失页面、断链 |

## 关键文件

- `主页.md` —— 分类目录，每页一行链接+摘要
- `log.md` —— 按时间顺序的追加日志（grep 可用格式：`## [YYYY-MM-DD] 操作 | 标题`）

## 灵感来源

与 Vannevar Bush 的 [Memex](Memex.md) (1945) 精神相通——个人策划的知识存储，文件之间的关联路径与文件本身同样珍贵。Bush 没解决的维护问题，LLM 解决了。
