# 链式思考 CoT

Chain of Thought——通过在 prompt 中加入中间推理步骤，引导 LLM 分步思考的技术。

## 工作原理

普通 prompt 直接要答案，CoT 要求在得出答案前展示推理过程：

```
普通："Roger有5个球，买2罐(每罐3个)，现在几个？→ 11"

CoT： "1. Roger初始有5个 2. 2罐共2×3=6个 3. 总数5+6=11 → 答案是11"
```

零样本 CoT 只需加一句：**"让我们一步一步思考"** 或 **"Think carefully and logically, explaining your answer."**

## 为什么有效

大语言模型的训练目标是预测下一个词，而不是逻辑推理。CoT 通过**强迫模型在生成"答案词"之前先生成"推理词"**，增加了正确路径被采样的概率。

## 从 CoT 到 Agent

| 演化 | 关键差异 |
|------|---------|
| CoT | 单次 prompt，线性链条，无分支 |
| ToT (思维树) | 多路径探索 + 评估 + 剪枝，本质是 BFS/DFS Agent |
| GoT (思维图) | 图结构，支持聚合和反馈循环，与 LangGraph 理念一致 |
| Agent | 多轮对话 + 工具调用 + 工作流编排 |

**CoT 是思维规划的最小单元，ToT/GoT/Agent 都是在 CoT 基础上的架构扩展。**
