# 提示词工程

通过精心设计输入指令（Prompt），引导大语言模型输出高质量结果的技术和方法论。

## 核心理念

**Prompt 是 AGI 时代的"编程语言"，提示工程师 = AI 时代的程序员。** 无需修改模型参数即可显著提升模型性能（准确率提升 40%+），解决幻觉、答非所问等生产环境问题。

### 关键洞见

> 如果写 prompt 就可以达到基本要求，微调可以进一步提升；如果 prompt 不起作用，微调成功的可能性就很低。

提示工程比微调有更高的优先级和灵活性——微调会改变模型参数、降低通用能力、增加长期维护成本。

## CO-STAR 框架

设计 prompt 结构时覆盖六个维度：

| 维度 | 含义 | 示例 |
|------|------|------|
| **C**ontext | 任务背景 | "你是一个资深 Python 工程师" |
| **O**bjective | 目标任务 | "编写用户登录接口代码" |
| **S**tyle | 写作风格 | "模仿李白豪放风格" |
| **T**one | 语气态度 | "正式/幽默/善解人意" |
| **A**udience | 目标受众 | "面向 5 岁儿童" |
| **R**esponse | 输出格式 | "JSON / Markdown / 列表" |

## 生产级 Prompt 模板

```
# 1. 角色定义
你是一名[领域专家]，需利用[专业能力]完成任务。

# 2. 任务指示
请完成[具体任务]，要求：[要求1]、[要求2]

# 3. 上下文
场景：[具体场景]，限制：[约束条件]

# 4. 示例 (可选，复杂任务必加)
示例1：输入"xx" → 输出"yy"

# 5. 输入与输出格式
输入：[数据形式]，输出：[格式要求]
```

## 五大核心原则

| 原则 | 说明 |
|------|------|
| 简洁性 | 用最短语言传递核心需求 |
| 具体性 | 明确任务边界（格式、风格、范围） |
| 上下文完整 | 提供角色、场景等背景 |
| 无歧义 | 规避多义词和模糊表述 |
| 逻辑清晰 | 任务描述连贯，避免矛盾 |

## 技术演进

| 阶段 | 技术 | 本质 |
|------|------|------|
| 1 | 零样本 / 少样本 | 直接任务 + 示例 |
| 2 | [LLM-AI/链式思考 CoT](链式思考 CoT.md) | 在示例中加入推理步骤 |
| 3 | 自我一致性 | 多次运行 + 投票 |
| 4 | 思维树 ToT | BFS/DFS 多路径探索 + 剪枝 |
| 5 | 思维图 GoT | 图结构思维，聚合 + 反馈循环 |
| 6 | Agent | 多轮对话 + 工具调用 + 工作流 |

## 安全防御

- **System Prompt 屏障**：明确禁止角色诱导和提示泄漏
- **输入过滤**：关键词黑名单 + 语义检测
- **输出校验**：二次审核敏感内容
