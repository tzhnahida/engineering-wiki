# 统一因子图姿态推断框架

> 六篇姿态估计算法论文的数学统一：不是拼接，是因子图在 SO(3) 上的结构化推断。服务于[全身动捕与头显系统](全身动捕与头显系统.md)的算法选型和论文策略。

---

## 一、问题建模

### 1.1 姿态推断的因子图表示

15 节点动捕系统的姿态估计可以建模为因子图上的最大后验估计：

$$
P(q_{1:T}, \mathbf{B}(x) \mid \text{IMU}_{1:T}, \text{optical})
$$

因式分解为五类因子：

```
┌────────────────────────────────────────────────────┐
│                                                    │
│  ① 过程因子 (陀螺/Transformer)                       │
│     φ_gyro(q_t, q_{t+1}, ω_t)                      │
│     → 连接相邻时间步的姿态                            │
│                                                    │
│  ② 重力观测因子 (加速度计)                            │
│     φ_accel(q_t, a_t)                              │
│     → 约束 Roll/Pitch                              │
│                                                    │
│  ③ 地磁观测因子 (磁力计 + 空间磁场模型)                 │
│     φ_mag(q_t, m_t, B(x_t))                       │
│     → 约束 Yaw，B(x) 随空间位置变化                   │
│                                                    │
│  ④ 运动链因子 (骨骼刚性约束)                          │
│     φ_chain(q_t^i, q_t^j)                         │
│     → 相邻节点间旋转在骨骼系中恒定                     │
│                                                    │
│  ⑤ 光学参考因子 (标记点重投影)                         │
│     φ_optical(q_t, marker_t)                      │
│     → 绝对姿态约束，mm 级精度                         │
│                                                    │
└────────────────────────────────────────────────────┘
```

### 1.2 六篇论文在这个因子图上的贡献

每篇不是提供了一个完整算法，而是**改写了某一条或多条边的数学形式**：

| 论文 | 贡献的因子/边 | 数学实质 |
|------|:---:|------|
| **VQF** | ①+②+③，求解结构 | 联合优化 → 条件独立的三段级联，磁干扰不污染倾斜 |
| **Correntropy GD** | ②+③，势函数 | $\phi(e)=e^2$ → $\phi(e)=\exp(-e^2/2\sigma^2)$，IMU 噪声是重尾分布 |
| **MDR** | ③，因子参数化 | 全局地磁北 → 空间函数 $\mathbf{B}(x,y,z)$，体素数据库 |
| **DO IONet** | ①，**取消态** | 积分模型 → 窗口直接映射 $p_\theta(q_t\mid\text{IMU}_{t-W:t})$ |
| **AVNet** | ②+③，生成机制 | 手工 $h(q)$ → 网络学出的 $h_\theta(q)$ + 协方差 $R_\theta$ |
| **Matrix Fisher** | 所有边的消息类型 | 点估计 $q$ → SO(3) 概率分布 $p(\mathbf{R}\mid\mathbf{F})$ |

## 二、统一推断算法

### 2.1 消息类型：Matrix Fisher 分布

所有因子之间的消息统一为 Matrix Fisher 分布：

$$
p(\mathbf{R} \mid \mathbf{F}) = \frac{1}{c(\mathbf{F})} \exp(\mathrm{tr}(\mathbf{F}^\top \mathbf{R})), \quad \mathbf{R} \in SO(3), \mathbf{F} \in \mathbb{R}^{3\times3}
$$

共轭性质：后验参数 = 先验参数 + 各似然参数之和。

### 2.2 每步推断

```
For each timestep t:

  1. [过程因子 ①]
     若 Transformer 可用:
       q_t^prior ← DO IONet(IMU_{t-W:t})          ← 跳过积分，直出姿态
     否则 (ESP32):
       q_t^prior ← q_{t-1} ⊗ exp(ω_t Δt)          ← VQF strapdown
     → 先验 Matrix Fisher: F_t^prior

  2. [重力观测 ②]
     ŷ_accel ← AVNet_accel(IMU_t)                  ← 网络学出伪测量
     R_accel  ← AVNet_cov(IMU_t)                   ← 网络学出协方差
     → 似然 Matrix Fisher: F_t^accel, 用 MKCL 匹配
       φ_accel = exp(-||ŷ_accel - h_θ(q)||²_{R}/2σ²)  ← Correntropy 核截断

  3. [地磁观测 ③]
     B_local ← query_MDR_database(pos_t)           ← MDR 体素查表
     → 似然 Matrix Fisher: F_t^mag

  4. [因子融合 (Matrix Fisher 共轭)]
     F_t^fused = F_t^prior + F_t^accel + F_t^mag

  5. [运动链消息传播 ④]
     For each neighbor j of node i:
       F_ij ← compute_chain_message(q_t^i, q_t^j, bone_R^{ij})
     F_t^i ← F_t^fused + Σ_j F_ji

  6. [光学参考 ⑤]
     If optical_available:
       F_t^i ← F_t^i + F_t^optical

  7. [空间磁场更新 (IAI 准则)]
     If mag_consistent(current, stored):
       B(pos_t) ← running_average(B(pos_t), m_t)

  8. [输出]
     R̄_t ← argmax_R p(R | F_t)                    ← 最可能姿态
     σ_t ← concentration(F_t)                      ← 不确定性
```

### 2.3 关键数学细节

**MKCL + InEKF 测量更新**（理论创新点）：

标准 InEKF 测量更新假设高斯似然 → 后验可解析闭合。当似然用 MKCL（非高斯，核截断），后验不再有闭合形式。需要：

- 变分推断：最小化 $KL(q(\mathbf{R}) \| p(\mathbf{R} \mid \text{obs}))$，约束在 Matrix Fisher 族内
- 或重要性采样：用先验采样，重加权近似后验

这是 Correntropy KF 文献（李士雷组）与 InEKF 文献（Barrau/Bonnabel）从未交叉的地方。

**跨节点 Transformer 注意力**：

DO IONet 只在单 IMU 的 200 帧窗口内做自注意力。扩展到 15 节点：
- 每个节点的 query 不仅 attend 自己的 key，也 attend 相邻节点的 key
- 注意力 mask 注入骨骼结构先验（肩-肘-腕三点的运动相关性高于臂-腿）
- 输出不再是单节点姿态，而是**联合条件分布**

## 三、精度与性能

### 3.1 渐进精度

| 层级 | 配置 | 单节点 Yaw | 关节角度 RMSE | 部署 | 对标 |
|------|------|:---:|:---:|:---:|------|
| L0 | Madgwick 单节点 | 3.7° | ~5° | ESP32 | SlimeVR |
| L1 | VQF 解耦 + MKCL | **2.9°** | ~3° | ESP32 | — |
| L2 | L1 + 运动链因子 | **~1°** | **~1.5°** | ESP32/PC | Perception Neuron |
| L3 | L2 + 跨节点 Transformer | **<0.5°** | **~0.8°** | PC GPU | **Rokoko** |
| L4 | L3 + InEKF + MKCL 测量 | **<0.3°** | **~0.5°** | PC GPU | Xsens(无光学) |
| L5 | L4 + MDR + 光学参考 | **<0.15°** | **<0.3°** | PC GPU+光学 | **接近 OptiTrack** |

### 3.2 计算性能

**Phase 1 · ESP32-S3 边缘（L0–L2）**

| 组件 | 每步耗时 | 1kHz 负载 |
|------|:---:|:---:|
| VQF Basic | ~50μs | ~5% |
| VQF 完整 + MKCL | ~100μs | ~10% |
| MDR-lite 体素查询 | ~50μs | ~5% |
| **合计** | **~150μs** | **~15%** |

> ESP32-S3 240MHz 双核，1 核 AHRS + 1 核 WiFi/CAN，远不到瓶颈。

**Phase 2 · PC GPU（L3–L5）**

| 组件 | 延迟 | 内存 |
|------|:---:|:---:|
| Transformer 编码器 | ~0.5ms | 5MB |
| 跨节点注意力 | ~3ms | 20MB |
| AVNet 协方差适配器 | ~1ms | 3MB |
| InEKF 15 节点 | ~0.5ms | 1MB |
| Matrix Fisher 融合 | ~2ms | 5MB |
| **总延迟** | **~7ms** | **~35MB** |

> ~140Hz 等效帧率，远超 90Hz 目标。训练 DO IONet 需 ~4h / 4GB VRAM，离线一次性。

## 四、算法创新评估

### 4.1 真正的算法创新

**唯一一个有理论深度的数学贡献：MKCL + InEKF 在 Lie 群上的测量更新。**

相关熵 KF 文献（李士雷，港科大）在欧氏空间完成——标准 KF 框架，状态是 $\mathbb{R}^n$。InEKF 文献（Barrau/Bonnabel）在 SO(3) 上使用高斯假设。两者从未交叉。

将 MKCL 嵌入 InEKF 需要解决的核心问题：Matrix Fisher 先验 × MKCL 似然 → 后验不是 Matrix Fisher（核函数破坏了共轭性）。需要变分近似或重要性采样闭合。这是 Lie 群上非高斯滤波的新理论结果。

### 4.2 有贡献但非纯算法的点

| 贡献 | 分类 | 强度 |
|------|------|:---:|
| 跨节点 Transformer + 骨骼结构注意力 mask | 架构扩展 | 中等 |
| MDR 3D 磁建模 + 15 节点分布式协同建库 | 场景创新 | 中等 |
| 因子图在 SO(3) 上形式化六篇论文的统一 | 形式化贡献 | 中等 |
| 光学参考 + IMU 因子图联合优化 | 多模态融合 | 中等 |
| 完整系统的定量对标实验（vs 商用系统） | 工程贡献 | **强** |

### 4.3 论文定位

| 期刊/会议 | 适合吗 | 原因 |
|------|:---:|------|
| CVPR / ICCV | ❌ | 需要新网络结构、新 benchmark |
| IJRR / T-RO | ❌ | 需要严格的收敛性/可观测性证明 |
| **IEEE T-ASE** | ✅ | 系统级创新 + 定量对标实验 |
| **IEEE TIM** | ✅ | 精密测量系统验证 |
| **ISMAR / IEEE VR** | ✅ | VR/AR 动捕应用 |
| **IEEE RAL** (短论文) | ✅ | MKCL+InEKF 可拆成 4-6 页理论短文 |
| Workshop | ✅ | 绰绰有余 |

## 五、论文策略建议

### 5.1 主论文

**目标期刊**：IEEE T-ASE 或 TIM

**卖点**：
- 完整的低成本高精度动捕系统（¥1366 vs $3k–15k 商用方案）
- 因子图统一框架——形式化六篇论文的贡献并将它们整合到同一推断器中
- MKCL + InEKF 在 Lie 群上的测量更新（数学深度）
- 充分的定量实验：对标 Xsens / Rokoko / Perception Neuron + 光学金标准

### 5.2 可选拆出

如果精力允许，将 MKCL + InEKF 单独写成 4–6 页理论短文投 IEEE RAL 或 CDC。
这是唯一有潜力发纯算法期刊的东西。

### 5.3 不要做的

- 不要声称"六篇论文的算法创新"——审稿人会逐条质疑 novelty
- 不要把系统完整性和算法创新混在一起推销——区分清楚，系统论文卖系统，算法短文卖算法
- 不要试图投 CVPR/ICCV 除非你有新的网络结构或大规模 benchmark
