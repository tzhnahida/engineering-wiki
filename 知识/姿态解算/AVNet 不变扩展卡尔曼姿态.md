# AVNet · 深度学习 + 不变扩展卡尔曼姿态融合

> CNN+GRU 从手机 IMU 学出姿态和速度，同时学出 Kalman 协方差——深度学习替代了所有人工调参。武汉大学 2025。

[📷 _llm/raw/assets/papers/avnet2025/avnet_p4_fig1.jpg|620]
*Fig. 1 — DMDVDR 框架总览：AVNet → 协方差适配器 → InEKF，端到端 IMU→3D 位姿*

## 问题设定：智能手机车载航位推算

智能手机导航在 GNSS 信号良好的场景下表现尚可，但进入**停车场、隧道**等 GNSS 拒止环境时定位精度急剧下降。消费级手机 IMU（如华为 Mate 30 搭载的 STM LSM6DSM）的精度远低于车载级：

- 陀螺仪速率噪声密度：**3.8 x 10^-3 deg/s/sqrt(Hz)**
- 加速度计噪声密度：**g x 90 x 10^-6 /sqrt(Hz)**（g 约 9.80 m/s^2）

纯 IMU 机械编排在数分钟内即可产生数十米的定位偏差。车辆运动虽然有先天约束（横向/垂向速度近似为零、前进方向有里程约束），但智能手机**无法直接读取车载里程计或转角传感器**——这是 AVNet 方法的基本动机：用深度学习虚拟出这些传感器。

> [!note] 与传统纯 IMU 积分的本质区别
> 传统航位推算用 IMU 原始数据做机械编排（积分得姿态→速度→位置），误差随时间三次方增长。AVNet 的方案是：网络从 IMU 时序中**直接回归**出姿态和速度作为"伪观测"，Kalman 滤波再用这些观测去校正机械编排——网络替代的是物理传感器，而不是替代滤波模型。

## AVNet 网络：CNN-GRU 混合架构

### 输入与输出

| 参数 | 值 |
|------|-----|
| 输入窗口 | 200 帧 IMU 原始数据（200 x 6：三轴加速度 + 三轴角速度） |
| 输出频率 | 1 Hz（每 200 帧输出一次） |
| DDATT（姿态） | 3-DOF 姿态变化量（四元数 xyz 分量），需累加得绝对姿态 |
| DDODO（速度） | 1-DOF 前进方向速度 |
| 实现 | PyTorch，开源（GitHub: DragonEmperorG/QDeepOdo） |


### 网络结构细节

```
输入 (200, 6)
    ↓
1D Conv (5x1) + ReLU + MaxPool
    ↓
1D Conv (5x1) + ReLU + MaxPool
    ↓
全连接层 x 2
    ↓
GRU 模块
    ↓
输出: 姿态变化 Δq(x,y,z)  或  前进速度 v_lon
```

- **CNN 部分**：两层 1D 卷积（kernel=5, ReLU + 最大池化）提取局部时序特征。第一层输入 6 维输出 32 维，第二层维持 32 维
- **GRU 部分**：门控循环单元建模长程依赖，相比 LSTM 参数量更少，适合手机端推理
- **姿态分支与速度分支**：共享相同网络结构，独立训练
- **姿态学习目标**：输出四元数的 x, y, z 分量表示的**姿态变化量**（而非绝对姿态），避免奇点。损失函数为预测 Δq 与真值 Δq 的 MSE。初始绝对姿态需外部给定
- **速度学习目标**：输出前进方向速度标量，损失函数为 MSE。训练中速度真值来自 NovAtel SPAN 参考系统

### 协方差适配器（Filter Parameter Adapter）

AVNet 的配套网络——一个更简单的 2 层 CNN——实时输出过程噪声 Q 和测量噪声 N 的对角元素：

- **输入窗口**：20 帧（更短窗口匹配 200 Hz 输出频率）
- **输出结构**：Q 为 6 个 3x3 对角块（对应姿态、速度、位置、陀螺偏置、加计偏置、外参等），N 为 2 个 3x3 对角块（DDATT + DDODO/DDNHC）
- **动态范围**：参数 beta=3 使得每个协方差可在默认值的 10^3 倍到 10^{-3} 倍之间调节
- **训练策略**：间接优化——适配器输出接入 InEKF，以**相对平移误差**为损失函数端到端训练。这使得协方差参数更接近其**数学意义**（滤波不确定性）而非物理意义

> [!note] 协方差适配器的设计哲学
> 传统 Kalman 滤波的 Q 和 R 是固定超参。适配器让网络根据当前 IMU 信号质量**实时调节**测量噪声——IMU 信号可信时降低 R（信任伪观测），可疑时升高 R（依赖模型预测）。这个 "learned uncertainty" 思路与 [AI-IMU Dead Reckoning](AI-IMU Dead Reckoning.md) 一脉相承，但 AVNet 的适配器输出维度更丰富（覆盖所有状态 + 测量噪声）。

## InEKF 融合框架：为什么是李群方法

### 标准 EKF 在旋转估计上的缺陷

标准 EKF 将状态误差定义为欧几里得空间中的向量差（如四元数之差 delta-q），这破坏了 SO(3) 的流形结构：

- 需要频繁的四元数归一化（额外数值步骤）
- 协方差传播在强非线性区域失真
- 收敛域有限——初始误差大时容易发散

### 不变 EKF 的数学基础

InEKF 将系统状态嵌入李群 SE2(3)（扩展欧几里得群）：

```
χ = [R  v  p]   ∈ SE2(3)
    [0  1  0]
    [0  0  1]
```

完整滤波器状态包含五个变量：

```
x = { χ_w_s, δω, δf, R_v_s, p_s_v }
```

其中 χ_w_s 是传感器在世界系下的位姿（含姿态、速度、位置），δω/δf 是 IMU 偏置，R_v_s 是传感器→车体外参，p_s_v 是杆臂。

**误差定义（右不变误差）**：

```
η = χ̂ χ⁻¹ = exp(ξ)    (ξ ∈ 𝔰𝔢₂(3) 为李代数向量)
```

展开为：
```
η_R = R̂ Rᵀ = exp(ξ_R)          （姿态误差，在 𝔰𝔬(3) 上）
ξ_v = v̂ - R̂ Rᵀ v               （速度误差，与姿态耦合）
ξ_p = p̂ - R̂ Rᵀ p               （位置误差，与姿态耦合）
```

偏置误差和外参误差定义为**标准线性误差**（非不变）：

```
ξ_δω = δω - δω̂,    ξ_δf = δf - δf̂
η_Rv = R̂_v_s (R_v_s)⁻¹     （在 SO(3) 上）
ξ_psv = p_s_v - p̂_s_v
```

### 群仿射性质——InEKF 的杀手锏

系统动力学 f(χ) 满足**群仿射**（group affine）条件：

```
f(χ₁χ₂) = χ₁ f(χ₂) + f(χ₁) χ₂ - χ₁ f(I) χ₂
```

当系统满足此性质时，**误差动力学与状态轨迹无关**——无论系统当前在什么状态，误差的线性化矩阵是相同的。这意味着 InEKF 的**收敛性和一致性不依赖于当前估计的精度**，具备类似线性 Kalman 滤波的全局性质，这是标准 EKF 无法保证的。

### 右不变 vs 左不变误差

论文选择了**右不变误差** η = χ̂ χ⁻¹。选择依据：

- 右不变误差使姿态误差的线性化从当前姿态估计中解耦（Jacobian 中 R̂ 出现在偏置传播通道而非姿态误差本身）
- 对于 IMU 惯导这类**前向传播过程模型已知、测量模型在输出空间**的问题，右不变误差的 Jacobian 更稀疏
- 左不变误差 η = χ⁻¹ χ̂ 也有类似性质，但相应的测量模型 Jacobian 结构不同——两种选择在理论上等价，实践中取决于测量模型的形式

### 伪观测的测量模型

**DDATT（数据驱动姿态）：**
```
y_R = R · exp(n_R)    （乘法噪声模型，符合 SO(3) 结构）
```

**DDODO + DDNHC（速度 + 非完整性约束）：**
```
y_v = [0, v_lon, 0]ᵀ + n_v
    其中 v_lon 由 AVNet 给出，lat=0、up=0 为 NHC 约束
```

速度测量中关键的**杆臂补偿**公式：
```
v_v = R_v_s · (R_s_w · v_w + [ω]× · p_s_v)
```

物理含义：车体坐标系下的速度 = 外参旋转 x (世界系速度转到传感器系 + 杆臂引起的切向速度)。这个公式同时在滤波器中估计外参 R_v_s 和杆臂 p_s_v——不需要事先精确标定。

测量残差与 Jacobian H 矩阵的维度为 6（3 姿态 + 3 速度），论文给出了完整的解析推导（公式 70-80）。

> [!note] AVNet 相对于 AI-IMU 的关键改进
> AI-IMU 仅使用 2D 速度约束（NHC + 简单速度模型），在消费级 IMU 上迅速发散——静止阶段的 ZUPT/ZIHR 无法抑制陀螺漂移。AVNet 加入 3D 姿态约束后，系统获得 6-DOF 连续观测更新，车辆静止和运动时都有修正信号。实验结果确认：陀螺误差是航位推算误差的主要来源，姿态约束是关键改进。

## 实验结果

### 主干结果

| 指标 | AI-IMU | DeepOdo | DeepOri | **AVNet (Proposed)** |
|------|--------|---------|---------|---------------------|
| 相对水平平移误差 Ethor | 11-66% | 8-22% | 4-13% | **0.29-0.58%** |
| 相对旋转误差 Errel (deg/m) | 50-571 | 12-160 | 8-56 | **1.0-2.1** |
| 相对平移误差 Etrel | 15-66% | 9-41% | 5-44% | **0.84-3.40%** |

（数据来源：论文 Table 1，11 条序列的综合范围，因速度和轨迹形状而异。）

[📷 _llm/raw/assets/papers/avnet2025/avnet_p17_fig3.jpg]

### 关键发现

1. **AI-IMU 完全失败**：仅靠 NHC 约束无法抑制消费级 IMU 的传感器分散性。静止阶段积累的巨大误差导致滤波器启动即发散。序列 11 的 AI-IMU 相对平移误差高达 174,846%——基本不可用
2. **姿态约束优于速度约束**：DeepOri（+DDATT）整体优于 DeepOdo（+DDODO），因为**陀螺误差**是主要误差源，姿态约束直接限制陀螺漂移
3. **两者缺一不可**：DeepOdo 在急转弯处航向发散（无姿态约束），DeepOri 在长距离下位置累积误差（无速度约束）。合并后 0.4% 水平误差
4. **隧道泛化能力**：GSDC 2023-2024 数据集的 578m 隧道段，55 秒 GNSS 完全中断，最终平移漂移仅 0.64%——接近停车场水平的性能，证明方法对陌生场景有泛化能力
5. **超低速鲁棒性**：5 km/h 下车速对比方法的性能显著恶化，而 AVNet 保持稳定——因为 DDATT + DDODO 提供**连续约束**，不依赖运动/静止状态检测

### 失效与局限性

- **频繁转弯**：位置误差加速累积——论文明确承认每次 90 度转弯都会引入额外漂移
- **安装约束**：智能手机固定于车体（刚性安装），未测试手持、口袋、车载支架等灵活安装
- **数据集规模**：单一手机型号（华为 Mate 30），同一车辆，共 11 条序列——与小样本学习相关的泛化性存疑
- **垂直方向**：固定重力假设导致垂直方向误差大于水平方向（停车场实际有轻微坡度）
- **初始姿态**：需外部给定初始姿态用于积分累积——初始姿态误差会影响后续所有估计

## 对比：AVNet vs [DO IONet Transformer直接姿态](DO IONet Transformer直接姿态.md)

| | DO IONet | AVNet |
|------|---------|------|
| 网络架构 | CNN + Transformer | CNN + GRU |
| 输出 | 直接出姿态+速度 | 姿态变化量+前进速度+协方差 |
| 滤波器 | 无（纯端到端） | InEKF（网络作为测量前端） |
| 理论框架 | 纯深度学习 | 深度学习 + 不变 Kalman 混合 |
| 可解释性 | 低（黑箱端到端） | 较高（Kalman 部分可分析） |
| 应用场景 | 6-DOF 通用（行人/车辆） | 车载航位推算专用 |
| 关键约束 | 无显式运动约束 | DDATT + DDODO + DDNHC 6-DOF |

## 工程启示

对于采用类似 ZUPT 速度约束的惯性导航系统，AVNet 提供了一个可迁移的混合框架思路：用深度学习从 IMU 时序中提取**连续的速度伪观测**，通过 InEKF 与 ZUPT 的零速约束、外部参考（视觉/光学）做统一融合。AVNet 证明消费级 IMU 在 GNSS 拒止下获得 0.4% 的定位精度是可行的——关键在于姿态与速度**同时约束**。

> [!note] 与 ZUPT 的关系
> 论文在讨论中明确指出：AVNet 的姿态和速度约束可以看作是 ZUPT 和 ZIHR 的**广义版本**——不仅在静止时工作，在运动时也持续提供约束。对于足部安装的 IMU，这种思路同样有价值：静止时 ZUPT 提供绝对参考（零速），运动时 AVNet 风格的速度估计提供连续约束，InEKF 统一融合。
