# AI 精准文献检索方法

## 核心原则

文献检索的目标不是"找到相关论文"，而是**证明你的贡献没有人做过**。

检索策略决定论文定位——搜得太宽，你的创新被淹没；搜得太窄，审稿人认为你文献调研不充分。

---

## 1. 检索之前：先定义你的学术坐标

### 常见错误

把**传感器清单**当成检索词：

```
❌ UWB + IMU + 气压计 + ZUPT
   → 几千篇，你的论文是其中一篇
```

### 正确做法

用**方法特征**而非传感器列表来定位：

```
✅ 运动链约束 + 人体遮挡推断 + 差分气压测高
   → 别人搜不到这个组合，但你搜到的每一篇都高度相关
```

### 原则

> **关键词描述的是你的方法，不是你的传感器清单。**

---

## 2. 中外文数据库检索策略

### 中文数据库（CNKI、万方、维普）

| 步骤 | 操作 | 目的 |
|:---:|------|------|
| 1 | 用 2–3 个领域通用术语建检索池 | 确认你在正确的文献簇里 |
| 2 | 叠加你的区分词，观察结果数变化 | 每加一个区分词，结果少一个数量级 |
| 3 | 全选同义词扩展 + 中英文扩展 | 避免遗漏变体表述 |
| 4 | 最终组合搜不到结果 → **你是第一个** | novelty 的直接证据 |

#### 实操示例

```
# Step 1: 建池子
惯性动作捕捉 + 扩展卡尔曼滤波
→ 几百篇，确认领域存在

# Step 2: 加区分词
惯性动作捕捉 + 误差状态卡尔曼滤波 + 运动链
→ 大幅缩小

# Step 3: 扩展搜
惯性动作捕捉 ∩ SO(3) ∩ 分布式姿态估计
→ 如果接近 0，你的方向是新的

# Step 4: 验证组合
UWB ∩ 差分气压计 ∩ 人体遮挡
→ 搜不到 = novelty 确认
```

### 英文数据库（IEEE Xplore、Google Scholar、Scopus、Web of Science）

| 数据库 | 优势 | 适用场景 |
|------|------|------|
| **Google Scholar** | 覆盖最广，引用追踪 | 初始探索，找"谁引了谁" |
| **IEEE Xplore** | 工程/传感器领域权威 | 查找对标论文 |
| **Scopus** | 结构化摘要，引文分析 | 系统性文献综述 |
| **Web of Science** | SCI 收录标准，影响因子 | 确认目标期刊的文献标准 |
| **arXiv** | 预印本，最新成果 | 查有没有人抢先 |

#### 英文检索式构造

```
# 不这么搜
"UWB" AND "IMU" AND "EKF" AND "indoor positioning"
→ 数千篇

# 这么搜
("kinematic chain" OR "body model") AND ("UWB occlusion" OR "body shadowing") AND "wearable"
→ 高度相关，可能 < 10 篇
```

## 3. 检索结果的分类处理

检索完成后，把所有结果分成四类：

| 类型 | 含义 | 怎么用 |
|:---:|------|------|
| **直接竞争者** | 方法高度相似 | 放进 Related Work，作为主要对比对象 |
| **邻域工作** | 不同场景但同方法 | 证明该方法在其他场景有效，但你的场景是新的 |
| **组件重叠** | 用了你的一部分组件 | 证明单个组件不新，但组合是新的 |
| **无重叠** | 完全不相关 | 忽略 |

### 关键认知

> 查到的论文多 ≠ 你的创新弱。恰恰相反——**每个组件都有人做过，但你的组合没人做过，这才是正常的创新形态。**

## 4. AI 辅助检索工具

| 工具 | 用途 | 注意 |
|------|------|------|
| **Semantic Scholar** | 语义搜索，引用图谱 | 免费，API 可用 |
| **Connected Papers** | 可视化文献关系图 | 适合探索 Related Work 网络 |
| **Research Rabbit** | 引文追踪，文献发现 | 从一篇种子论文扩展 |
| **CNKI AI 阅读** | 中文论文摘要提取 | 快速过滤不相关论文 |
| **ChatGPT/Claude** | 生成检索式，分析论文 | 不能替代检索，但能辅助分析 |

### AI 辅助的正确用法

1. 人工构造初始检索式
2. AI 帮你生成同义/变体检索式（中英文）
3. 用数据库执行检索（不用 AI 直接搜——AI 的论文库不全且滞后）
4. AI 帮你分析检索结果的差异和漏检

## 5. Novelty 验证 Checklist

提交论文前，确认以下问题：

- [ ] 你的**核心方法组合**在 CNKI 搜不到（中文确认）
- [ ] 在 Google Scholar / IEEE Xplore 搜不到（英文确认）
- [ ] 你能说出每一篇相关论文跟你的**本质区别**（不只是场景不同）
- [ ] 你的 Related Work 里引用的论文覆盖了检索结果的前 10 篇
- [ ] 你的 contribution 能用一句话区分于现有工作

## 6. 实例：从"没创新"到"确认新颖"

### 初始心态
> "UWB 有人做过，IMU 有人做过，ZUPT 有人做过，气压计有人做过——我没创新。"

### 检索验证

| 检索式 | CNKI 结果 | 结论 |
|------|:---:|------|
| UWB + IMU | 数千篇 | 组件烂大街 |
| UWB + IMU + ZUPT | ~2 篇 | 组合有人做过 |
| UWB + IMU + 气压计 | ~11 篇 | 组合有人做过 |
| **UWB + 气压计 + ZUPT** | **0 篇** | 三组件组合是新的 |
| **运动链 + UWB 遮挡 + 自适应协方差** | **0 篇** | 方法是新的 |

### 最终叙事

> 每个单一传感器技术都是成熟的。三传感器组合在检索中没有形成子方向。更重要的是——**用运动链模型统一调度传感器的可信度**这一方法论，在现有文献中不存在。

## 参考

- CNKI 专业检索语法：`SU='主题词' * '主题词'` 为与，`+` 为或
- Google Scholar 高级搜索：`intitle:"keyword"` 限定标题
- IEEE Xplore：`"Abstract":keyword` 限定摘要
- 同义词扩展：中文开"中英文扩展"，英文加同义词用 `OR` 连接
