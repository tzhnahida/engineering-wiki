# 脚装惯性导航与 ZUPT 奠基

> Foxlin 2005 — 首次完整提出鞋装 IMU + 软 ZUPT + EKF 的行人追踪系统。不是"归零速度"而是"把零速当作 Kalman 测量"——这一步让 EKF 能反向修正步态期间的漂移。对后续 Skog 2010 的 GLRT 形式化有直接奠基作用。

---

## 为什么是奠基

| | 传统方案 | Foxlin 2005 |
|------|------|------|
| ZUPT 方式 | 硬归零——速度=0、位置冻结 | **软 ZUPT**——静止时 EKF 收到速度=0 的测量，协方差自动传播到位置/姿态 |
| 静止期检测 | 无标准方法 | 多传感器阈值法：陀螺+加计偏离滑动均值超过阈值 → 清除 stillTime |
| 姿态约束 | 硬归零时姿势也被冻结 | 只约束速度，**不碰姿态** |
| 误差模型 | 无 | 15 维 EKF 误差状态：[φ δω δr δv δa] |
| 航向处理 | 无或简单磁力计 | 磁力计+磁偏角在线校准+GPS 传递对准 |
| 验证 | 无 | Vicon 参考：~2% DT；户外闭环 741m 误差 ~2m (<0.3%) |

## 系统：NavShoe

```
鞋装 InertiaCube3 (陀螺+加计+磁力计, 300Hz)
  → RF 无线 (50mA @5V)
  → 步态检测 (角速度能量 + 加计幅值阈值)
  → 静止期: EKF 测量更新 (v=0, ω=0 可选)
  → 非静止期: 捷联惯导积分 (Strapdown INS)
  → 可选: GPS 位置修正 + 磁力计航向修正
  → 输出: 位置 + 姿态 + 速度 (15 维 EKF)
```

## EKF 误差状态模型

### 状态向量

$$\delta x = [\phi^T \quad \delta\omega^T \quad \delta r^T \quad \delta v^T \quad \delta a^T]^T$$

| 分量 | 含义 | 维度 |
|------|------|:---:|
| φ | 姿态误差（欧拉角小角度） | 3 |
| δω | 陀螺零偏 | 3 |
| δr | 位置误差 | 3 |
| δv | 速度误差 | 3 |
| δa | 加计零偏 | 3 |
| **总计** | | **15** |

### 状态转移矩阵

$$\Phi = \begin{bmatrix}
I & -C_b^n \Delta t & 0 & 0 & 0 \\
0 & I & 0 & 0 & 0 \\
0 & 0 & I & I\Delta t & 0 \\
-S(f^n)\Delta t & 0 & 0 & I & C_b^n \Delta t \\
0 & 0 & 0 & 0 & I
\end{bmatrix}$$

$C_b^n$ 是体坐标系到导航坐标系的旋转矩阵。子块 $(\delta v, \phi)$ 中 $S(f^n)\Delta t$ 是这篇论文最精巧的设计——它让姿态误差通过比力 $f^n$ 传播到速度误差，从而在协方差矩阵中建立姿态和速度的**相关性**。这就是软 ZUPT 能反向修正姿态的原因。

### ZUPT 测量模型

静止时，EKF 收到**虚拟测量** $z = [0, 0, 0]^T$（INS 的当前速度估计为"零"）。

$$\nu = z - h(\hat{x}) = -v^{INS}$$

测量矩阵直接挑出速度误差分量：

$$H = [0 \quad 0 \quad 0 \quad I \quad 0]$$

对应的残差 $\nu$ 就是 INS 此时速度的负值。

### 为什么软 ZUPT 能反向修正步态漂移

硬 ZUPT 只能"本次静止时修正一次位置"。软 ZUPT 让 EKF **利用协方差矩阵中的相关性**：步态结束时，北向速度误差与北向位置误差高度相关 → 速度测量更新后，位置和姿态自动被拉回。这是 EKF 区别于简单重置的关键。

### 测量噪声的自适应设计

$$R = I \times \max(\text{trace}(HPH^T), (1\text{ mm/s})^2)$$

R 不小于当前速度协方差，确保 ZUPT 修正**渐进施加**（0.15-0.5 秒），避免一次性重置造成数值不稳定。

## 零速检测

```python
# 伪代码还原
stillTime += dt
if abs(gyro - gyro_avg) > THRESH_GYRO or abs(accel - accel_avg) > THRESH_ACCEL:
    stillTime = 0
    gyro_avg, accel_avg = update_average()

# 开始 ZUPT 的条件
if stillTime > 0.15:  # 持续静止 150ms
    apply_ZUPT()       # 持续到脚再次移动
```

阈值标定目标：步态期 `stillTime` 永不超过 0.1s，支撑期始终超过 0.3s。跑步需要更高阈值（触地时加计冲击尖峰）。

## 磁力计航向修正

### 磁力计误差模型

$$\text{Compass reading} = \text{true heading} + D + d + \text{noise}$$

| 误差源 | 来源 | 特性 |
|------|------|------|
| D (磁偏角) | 地磁轴与地理北极夹角 | 全球平滑（IGRF 模型），几公里内恒定 |
| d (磁偏差) | 局部地质/人文干扰 | 室内可达几十度，随位置快速变化 |
| noise | 传感器白噪声 | 工厂校准残差 ~0.1° |

### 每步取一次航向测量

**关键设计**：每个步态周期只在**支撑期末**取一次磁力计航向测量——此时脚即将离地，pitch/roll 经过一串 ZUPT 已修正到最精确水平。相邻两步的磁偏差 d 因空间位置不同而快速**去相关**，避免有色噪声问题。

> 传统做法是做足卡尔曼滤波器，额外维护一个 d 的状态来估计局部磁偏差。Foxlin 的做法更简洁——利用步态的**空间去相关**自然解耦连续测量，省掉一个状态变量。

## GPS 深度耦合

### 磁偏角在线校准

GPS 可用时：
1. GPS 位置修正 → 传递对准校正 yaw 和 yaw 陀螺零偏（精度 <1°）
2. 同步接受磁力计航向测量，扩展 KF 状态加入磁偏角 D
3. **自动校准**：GPS 传递对准的真实航向 − 磁力计读数 = D
4. GPS 丢失后，用校准的 D 修正磁力计，航向精度不退化

### 位置修正策略

GPS 噪声协方差设为 $(20\text{m})^2 \cdot I_{3\times3}$（WAAS 精度 ~6m，考虑时间相关性额外放大）。GPS 位置修正通过传递对准间接约束 yaw→yaw 陀螺偏置。

## 实验结果

### 室内
- Vicon 光学参考验证
- 纯惯导 + ZUPT（无磁力计、无 GPS）
- 精度：~2% 行走距离
- 高度漂移：每 100m 垂直 ±6cm

### GPS 辅助
- GPS 可用时：绝对精度 ~6m（GPS 限制）
- GPS 中断后：可维持几公里精度不崩溃（普通 GPS/INS 只能维持几秒）
- 磁偏角自动校准：GPS 在线标定 → 进入室内后自动继承真实北坐标系

## 与后续工作的比较

| | Foxlin 2005 | Skog 2010 | Wagstaff 2017 |
|------|:---:|:---:|:---:|
| ZUPT 检测 | 启发式多传感器阈值 | **GLRT 统计框架** | SVM 动作分类+自适应 |
| 检测器类型 | 1 种（角速度+加计联合） | 4 种（SHOE+退化） | 继承 Skog 检测器 |
| 阈值 | 固定标定 | 理论推导 γ | 动作自适应 (×14 跑步/走路) |
| 陀螺 | InertiaCube3 (战术级) | ADIS16355 (工业级) | 未指定 |
| 磁力计 | 磁偏角在线校准 | 不涉及 | 不涉及 |
| GPS | 深度耦合传递对准 | 不涉及 | 不涉及 |
| EKF 维度 | 15 | 15 | 15 |
| 验证 | Vicon + 户外闭环 | 数值仿真 + PSD | 5.9km 户外 6 动作 |

> Skog 2010 开篇写道 "Foxlin 第一个提出鞋装 IMU + ZUPT 的完整方案"。两者的关系：Foxlin 发明了方向盘（软 ZUPT + EKF 架构），Skog 给方向盘装了仪表盘（GLRT 统计检验 + 四种检测器对比），Wagstaff 给方向盘上了智能驾驶辅助（SVM 自适应阈值）。
