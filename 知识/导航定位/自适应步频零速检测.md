# AWGF-ZVD · 自适应步频零速检测

> 把 GLRT 的固定阈值变成步频的函数。走路和跑步不再是同一个阈值。南京航空航天大学 2024。

## 问题

传统 ZUPT 用 GLRT（广义似然比检验）检测零速区间：加速度+角速度的加权平方和超过阈值 T 就判为运动。

**固定阈值 T 的致命问题**：走路时测出的 T=3σ 刚好。跑步时加速度噪声变大，同样的 T 会漏检（脚已着地但检测器没触发）。放宽 T 则走路时的静止段被截短。

## 核心思路

**T 不应该是常数，应该是步频 f 的函数。**

```
传统:  if GLRT > T_fixed  → 运动
AWGF:  if GLRT > T(f)    → 运动  (f = 当前步频)
```

### 三步走

1. **自适应滑动窗口提取步频**
   - 对陀螺 Y 轴做滑动窗口 FFT
   - AMPD（自适应多尺度峰值检测）找到频谱主峰 → 步频 f
   - 窗口大小随 f 自适应：f 高（跑）→ 窗口小（快响应），f 低（走）→ 窗口大（频率分辨率高）

2. **三次多项式阈值曲线**
   - 采集不同步频下的最优 GLRT 阈值
   - 拟合：`T(f) = a·f³ + b·f² + c·f + d`
   - a/b/c/d 论文已给，直接用

3. **实时自适应**
   - 每步更新 f，查 T(f)，用新阈值做 GLRT

## 关键数据

- 优于所有固定阈值方法（精度 + 误检率双赢）
- 适用慢走/快走/跑步，无需重调参数
- 计算量低：FFT + 三次多项式，适合 MCU

## 与传统 ZUPT 的关系

**不是替代品——是增强层**。AWGF-ZVD 替换的是 ZUPT 的**检测器部分**（什么时候判静止）。修正部分（速度归零 + EKF 更新）保持不变。可以和任何 EKF/ESKF 搭配。

## 对本项目的意义

脚部节点的 ZUPT 目前用 [[2026-06-24 - Skog 零速检测算法评估]] 中的固定阈值方法。换上 AWGF-ZVD 后，慢走和跑步的零速检测都会更准。改动量 ~50 行。
